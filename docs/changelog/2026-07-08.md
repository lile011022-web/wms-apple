## Inbound Records Last-six Search

- `apps/api/src/modules/inbound/inbound.repository.ts`
  - 入库记录列表的 `search` 参数新增 IMEI/Serial 后 6 位末尾匹配。
  - 普通搜索仍覆盖包裹号、UPC、完整/局部 IMEI、Serial、父客户、子客户和商品信息；当输入正好为 6 位字母或数字时，额外按 `endsWith` 查找设备末尾号。

- `apps/api/src/modules/inbound/tests/inbound.repository.spec.ts`
  - 新增仓储层测试，锁定搜索框后 6 位会生成 IMEI/Serial 末尾匹配条件。

- `apps/web/src/pages/inbound-records/page.tsx`
  - 入库记录页搜索框提示更新为支持 `IMEI后6位`，让仓库操作员知道可以直接输入末尾 6 位查询。

- `docs/api/09-inbound-records.md`、`docs/product/05-inbound-rules.md`
  - 同步入库记录搜索规则：搜索框支持 IMEI/Serial 后 6 位定位。

### Usage Logic

1. 打开 `入库记录`。
2. 在搜索框输入 IMEI 或 Serial 的最后 6 位，例如 `123456`。
3. 点击 `查询` 后，系统返回 IMEI/Serial 以该 6 位结尾的入库明细；客户筛选仍可配合使用。

## Customer Alias / Sub-customer Aggregation

- `apps/api/prisma/schema.prisma`
  - 新增 `CustomerAlias`，用于在父客户下面维护 A1/A2/A3 这类收货名字。
  - `InboundBatch`、`InboundItem`、`InventoryItem` 新增可选 `customerAliasId`，保留入库来源子客户。
  - `customerId` 继续代表父客户和库存归属，保证客户库存、出库装箱和批量客户变更的主口径不被拆散。

- `apps/api/src/modules/customers`

- `apps/api/src/modules/inbound`
  - 创建入库草稿时可传 `customerAliasId`。
  - 后端校验别名必须属于当前父客户且处于启用状态。
  - 普通确认、强制入库、已确认记录修正创建库存时都会同步保留别名。

- `apps/api/src/modules/inventory`、`apps/api/src/modules/outbound`、`apps/api/src/modules/reports`

  - 入库、库存、装箱明细报表支持 `customerAliasId` 筛选，父客户总表和子客户分表可以分别下载。
  - 报表字段白名单新增 `customerAliasCode`、`customerAliasName`。

- `apps/web/src/pages/customer-management/page.tsx`
  - 客户管理页新增“子客户 / 收货别名”维护区，可在父客户下新增 A1/A2/A3。

- `apps/web/src/pages/inbound-scan/page.tsx`
  - 入库锁定区新增“子客户 / 别名”下拉；未选择时仍按父客户直接入库。
  - 刷新恢复入库草稿时会恢复已锁定的子客户。

- `apps/web/src/pages/detail-download/page.tsx`
  - 明细下载新增子客户筛选；选择父客户但不选子客户时下载父客户全量明细，选择子客户时下载该子客户来源明细。

- `apps/web/src/pages/outbound-packing/page.tsx`、`apps/web/src/pages/inbound-records/page.tsx`
  - 装箱可用库存、箱内明细和入库记录显示来源子客户，装箱归属仍保持父客户。

- `docs/product`、`docs/api`、`docs/database`
  - 同步父客户/子客户归集规则、入库别名规则、出库显示规则、报表筛选口径和数据库关系说明。

### Usage Logic

1. 在 `客户管理` 创建父客户 A。
2. 在 A 的“子客户 / 收货别名”区域新增 A1、A2、A3。
3. 入库时选择父客户 A，并按包裹收货名选择 A1/A2/A3；不需要拆成三个真实库存客户。
4. 确认入库后，库存归属仍是 A，但每条明细保留来源子客户。
5. 出库装箱选择 A，A1/A2/A3 的货都能一起进入 A 的箱子；页面会显示来源子客户。
6. 明细下载选择 A 可下载父客户全量；再选择 A1/A2/A3 可下载对应子客户来源明细。

## Customer Inventory SKU Deletion

- `apps/web/src/pages/customer-inventory/page.tsx`
  - 在客户库存的 SKU 汇总表新增行内删除按钮。
  - 删除前要求已选择具体客户；全部客户视图只用于查询，不能执行破坏性清理。
  - 确认删除时展示客户、SKU 和当前汇总数量，并按当前仓库、状态、日期筛选删除该 SKU 的库存行。

- `apps/api/src/modules/inventory`
  - `DELETE /inventory/products` 支持接收 `status`、`dateFrom`、`dateTo`，使 SKU 汇总行删除与当前页面筛选一致。
  - SKU 删除会继续保留客户、商品、UPC、入库单和入库历史，只清理匹配的库存行并断开库存指针。
  - 已装箱、已出库或仍有关联出库箱明细的库存会被后端拦截，避免破坏出库历史。

- `docs/product/06-inventory-rules.md`
  - 补充 SKU 汇总行删除规则：只能在选定客户后删除当前筛选范围内的库存行。

- `docs/api/10-inventory.md`
  - 更新 `DELETE /inventory/products` 请求体和安全约束。

## Usage Logic

1. 打开 `客户库存`，选择具体客户。
2. 根据需要选择仓库、日期范围和状态卡片，例如 `在库`。
3. 在 SKU 汇总表对应 SKU 行点击 `删除`。
4. 系统弹出确认框，确认后只删除当前筛选范围内该 SKU 的库存行。
5. 删除成功后库存汇总、SKU 汇总和明细列表会自动刷新。

## Google Sheets 订单补全同步

- `apps/api/src/modules/package-prealerts/google-sheets.client.ts`
  - 新增读取 `订单` sheet 的能力，默认 sheet 名称为 `订单`。
  - 新增读取和批量更新 `预报` sheet 行的能力，用于补全后回写原行而不是追加重复行。

- `apps/api/src/modules/package-prealerts/package-prealert-sheets-sync.service.ts`
  - 新增 `syncOrderEnrichment` 同步步骤。
  - 系统读取 `订单` sheet 后，按 `超链接` / `链接` / `订单链接` / `Apple订单链接` 匹配本地 Apple 订单预报。
  - 匹配成功后读取 `订单.单号` / `订单.物流单号`、`订单.预报仓库` / `订单.仓库`、`订单.名字` / `订单.姓名`，补全本地真实物流单号、仓库备注和姓名。
  - 本地 `APPLE-W...` 订单引用会在 `订单.单号` 或 `订单.物流单号` 有真实承运商单号时被替换，并自动识别 UPS/USPS/FedEx。
  - 补全后直接更新 `预报` sheet 中匹配的原行，不再新增重复行，并保留订单表提供的物流类型、预计交付日期、物流查询链接和订单状态。
  - `完整同步` 调整为 `写入预报 -> 读取订单补全并更新预报 -> 读取状态`。

- `apps/api/src/modules/package-prealerts/package-prealerts.controller.ts`
  - 新增 `POST /package-prealerts/integrations/sheets/orders`，用于手动读取 `订单` sheet 并补全预报。

- `apps/web/src/api/workflow.ts`, `apps/web/src/pages/package-prealerts/page.tsx`
  - 包裹预报页新增 `读取订单补全` 按钮。
  - Google 表格同步说明更新为 `写“预报”，读“订单”补全，再读“状态”回传`。

- `.env.example`, `.env.production.example`, `apps/api/src/config/env.schema.ts`
  - 新增 `GOOGLE_SHEETS_ORDER_SHEET_NAME=订单` 配置项。

- `docs/api/17-package-prealerts.md`, `docs/product/13-package-prealert-prd.md`
  - 同步三段式 Google Sheets 交换逻辑和字段规则。

### Usage Logic

1. 在 WMS 创建预报，Apple 订单链接会先写入 `预报` sheet。
2. 点击 `读取订单补全`，系统读取 `订单` sheet，按 Apple 链接匹配。
3. 如果 `订单` sheet 有真实 `单号` 或 `物流单号`，WMS 本地预报会从 `APPLE-W...` 更新为真实物流单号。
4. 系统把补全后的物流单号、姓名、仓库等字段更新回 `预报` 原行。
5. 仓库或对方系统后续把入库结果写到 `状态` sheet。
6. 点击 `读取状态` 或 `完整同步`，WMS 更新入库状态和异常预警。

### Usage Logic

1. WMS 只把预报写到 `预报`。
2. 对方系统把真实物流单号、物流状态、入库状态写到 `状态`。
3. WMS 点击 `读取状态补全` 后，按 `预报ID`、Apple 链接或物流单号匹配本地预报。
4. 匹配成功后补全本地单号和状态，并更新 Google `预报` 原行。
