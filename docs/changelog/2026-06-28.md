- Added automatic package-tracking-to-UPC focus transfer. When the package tracking input receives a complete scan value, the page moves focus to UPC without requiring Enter.
- Reset the automatic scan submission guard after failed add attempts and re-evaluate it after abnormal package tracking is confirmed, so a recoverable warning does not permanently block auto-add for the same scanned values.
- Fixed same-package continuous scanning so the retained package tracking number's current-draft duplicate warning is automatically treated as confirmed for the next item. Unsupported tracking formats and historical confirmed duplicates still require explicit continue confirmation.

## Usage Logic

- Operators can read the exact abnormal reason before deciding whether to correct or delete the row.
- UPC review and the current draft table use chronological review order: earliest scan first, latest scan last.
- Normal scanning returns focus to the package tracking input after each row. Same-package continuous scanning keeps the package tracking number and returns focus to UPC after each successful row.
- Package tracking scans now advance to UPC automatically, while abnormal package tracking still requires explicit continue confirmation before saving.
- Standard-mode scanning now advances from UPC to IMEI automatically once the UPC scan is complete.
- Auto-add is intentionally blocked when the same IMEI or Serial already exists in the active draft; operators must correct or delete the duplicate row before continuing.
- Browser validation covered standard scanning, abnormal tracking confirmation, duplicate IMEI blocking, tracking+UPC scanning, and same-package continuous scanning in both modes.
