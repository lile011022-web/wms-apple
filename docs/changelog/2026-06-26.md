## 9622 FedEx 长单号自动放行

### What Changed

- 入库扫码的物流单号自动放行规则从固定 `9622` + 18 位数字，调整为 `9622` 开头且总长 22 到 34 位的 FedEx 数字单号。
- 现场扫码到较长的 `9622...` FedEx 单号时，不再误提示“物流单号异常”，可以继续扫描 UPC/IMEI 并自动加入当前入库单。
- USPS、非 `9622` FedEx、其它不符合自动放行规则的物流单号，以及重复物流单号，仍然需要操作员手动确认。
- 入库扫码页的异常提示文案同步说明 `9622` FedEx 的 22-34 位规则。

### Touched Modules

- `packages/shared/src/validators/ups.ts`：维护 UPS、USPS、FedEx 物流单号格式校验和自动放行规则。
- `packages/shared/src/validators/scan.test.ts`：覆盖长 `9622` FedEx 单号自动放行。
- `apps/api/src/modules/inbound/tests/inbound.service.spec.ts`：验证入库物流单号检查接口接受长 `9622` FedEx 单号。
- `apps/web/src/pages/inbound-scan/page.tsx`：更新物流单号异常提示文案。
- `docs/product/05-inbound-rules.md`、`docs/api/08-inbound-scan.md`：同步产品规则和 API 合同。

### Usage Logic

1. 操作员先扫描物流单号。
2. 如果是 UPS，或 `9622` 开头且总长 22 到 34 位的 FedEx 数字单号，系统直接允许继续扫描。
3. 如果是 USPS、非 `9622` FedEx、格式不支持或重复单号，页面仍显示异常确认条。
4. 操作员确认异常后，当前单号才可以继续保存到入库草稿。

## 入库记录 UPC 修正

### What Changed

- 入库记录页新增 `修正UPC` 操作，已确认入库但 UPC 扫错的记录可以在原行直接修改。
- 保存修正时必须填写原因，系统会校验新 UPC 是否匹配启用商品。
- 修正成功后同步更新入库明细和对应库存行的 UPC、商品归属，客户库存、出库装箱和报表会看到修正后的商品。
- 如果库存已经装箱或出库，系统会阻止修正，避免已流转货物被静默改历史。
- 后端新增 `INBOUND_UPC_CORRECTION` 审计动作，记录修正前后 UPC、商品、库存状态和原因。

### Touched Modules

- `apps/api/prisma/schema.prisma`、`apps/api/prisma/migrations/20260626000000_inbound_upc_correction/migration.sql`：新增 UPC 修正审计动作。
- `apps/api/src/modules/inbound/dto/correct-inbound-record-upc.dto.ts`：定义入库记录 UPC 修正请求。
- `apps/api/src/modules/inbound/inbound.controller.ts`：新增 `PATCH /inbound/records/:id/upc`。
- `apps/api/src/modules/inbound/inbound.service.ts`：维护 UPC 修正校验、库存状态限制和商品匹配。
- `apps/api/src/modules/inbound/inbound.repository.ts`：事务更新入库明细、库存明细并写审计日志。
- `apps/api/src/modules/inbound/tests/inbound.service.spec.ts`：覆盖 UPC 修正成功和已装箱库存阻断。
- `apps/web/src/api/workflow.ts`、`apps/web/src/pages/inbound-records/page.tsx`：入库记录表新增 UPC 行内修正入口。
- `apps/web/src/styles/globals.css`：维护 UPC 修正输入框和操作按钮布局。
- `docs/product/05-inbound-rules.md`、`docs/api/09-inbound-records.md`：同步产品规则和 API 合同。

### Usage Logic

1. 操作员进入 `入库记录`，找到 UPC 错误的已确认记录。
2. 点击该行 `修正UPC`。
3. 输入正确 UPC 和修正原因。
4. 点击 `保存` 后，系统校验 UPC 商品映射并同步更新入库明细和库存。
5. 已装箱或已出库的库存不能在这里修正，需要先处理对应出库流程。
