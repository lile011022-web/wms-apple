## 9622 FedEx 长单号自动放行

### What Changed

- 入库扫码的物流单号自动放行规则从固定 `9622` + 18 位数字，调整为 `9622` 开头且总长 22 到 34 位的 FedEx 数字单号。
- 现场扫码到较长的 `9622...` FedEx 单号时，不再误提示“物流单号异常”，可以继续扫描 UPC/IMEI 并自动加入当前入库单。
- USPS、非 `9622` FedEx、其它不符合自动放行规则的物流单号，以及重复物流单号，仍然需要操作员手动确认。
- 入库扫码页的异常提示文案同步说明 `9622` FedEx 的 22-34 位规则。

### Touched Modules

- `packages/shared/src/validators/ups.ts`：维护 UPS、USPS、FedEx 物流单号格式校验和自动放行规则。

- `apps/api/src/modules/inbound/tests/inbound.service.spec.ts`：验证入库物流单号检查接口接受长 `9622` FedEx 单号。

- `docs/product/05-inbound-rules.md`、`docs/api/08-inbound-scan.md`：同步产品规则和 API 合同。

### Usage Logic

1. 操作员先扫描物流单号。
2. 如果是 UPS，或 `9622` 开头且总长 22 到 34 位的 FedEx 数字单号，系统直接允许继续扫描。
3. 如果是 USPS、非 `9622` FedEx、格式不支持或重复单号，页面仍显示异常确认条。
4. 操作员确认异常后，当前单号才可以继续保存到入库草稿。

- `docs/product/07-outbound-rules.md`、`docs/product/10-report-rules.md`、`docs/api/11-outbound-packing.md`、`docs/api/14-reports.md`：同步未封箱单箱数据下载规则。

### Usage Logic

1. 仓库在出库装箱页面创建箱子并加入货物。
2. 需要客服先出单时，点击 `下载全部数据` 下载当前客户/仓库装箱明细，或在对应箱子卡片点击 `下载数据` 下载单箱明细。
3. 系统生成装箱明细 Excel 并下载，箱子仍保持原状态。
4. 后续真正封箱时，仍需上传照片或视频凭证后才能点击确认封箱。

## 已创建箱子按任务表头分组

### What Changed

- 出库装箱页的 `已创建箱子` 区域新增任务分类按钮和分组段落。
- 系统会从可见箱名里自动提取最后 `箱{数字}` 前面的内容作为任务表头。
- 例如 `wangchen20260625翻 新ipad箱7` 会归到 `wangchen20260625翻 新ipad`，`wangchen20260625翻 新手机 16pro箱1` 会归到另一组。
- 已创建箱子列表每页加载数量调到 50 个，让当天多个任务的大部分箱子可以在同一页内完成分组。
- 分组只影响页面查找和浏览，不改变箱号、箱子状态、删除、设为当前箱、下载数据或封箱逻辑。

### Touched Modules

- `apps/web/src/pages/outbound-packing/page.tsx`：按箱名任务表头分组渲染已创建箱子，并增加任务筛选按钮。
- `apps/web/src/styles/globals.css`：增加任务分组按钮、分组标题和分组列表样式。
- `docs/product/07-outbound-rules.md`、`docs/api/11-outbound-packing.md`：同步箱子分组规则和无需新增 API 字段的说明。

### Usage Logic

1. 仓库在出库装箱页创建或查看多个箱子。
2. 页面读取箱子显示名，自动把最后 `箱1`、`箱2`、`箱7` 前面的文本识别为任务表头。
3. 操作员可以点顶部任务按钮只看某一类，也可以保持 `全部` 查看按任务分段后的所有箱子。
4. 每个箱子卡片上的原有操作继续按单箱执行。
