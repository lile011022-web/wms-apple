## 包裹预报真实物流单号补齐后重推

- `apps/api/src/modules/package-prealerts/package-prealert-sheets-sync.service.ts`
  - 首次只有 Apple 订单链接时，`预报` sheet 仍只写链接，不把 `APPLE-W...` 当物流单号推送。
  - 当 `读取状态` 从对方回传中拿到真实 `物流单号` 后，系统会把本地预报从 `APPLE-W...` 替换成真实 UPS/USPS/FedEx 单号。
  - 如果该预报尚未入库，系统会把 `exchangePushStatus` 改回 `PENDING`，让下一次 `写入预报` 追加一行补齐真实物流单号的数据。
  - 二次 `写入预报` 时，`物流类型`、`物流单号`、`物流查询链接` 会使用真实承运商数据；Apple 订单引用不会写入 `物流单号` 列。

- `apps/web/src/pages/package-prealerts/page.tsx`
  - 明细表字段名保持为 `物流单号 / 订单引用`，区分 Apple 订单引用和真实承运商单号。

- `docs/api/17-package-prealerts.md`, `docs/product/13-package-prealert-prd.md`
  - 补充“链接预报 -> 对方补真实物流单号 -> WMS 读取并重推 -> 对方继续物流/入库状态回传”的内部闭环。

### 使用逻辑

1. WMS 首次创建预报并写入 `预报` sheet，只推 Apple 链接和基础信息。
2. 对方脚本从链接或对方数据中找到真实物流单号，并回写到 WMS 当前读取的状态表字段中。
3. WMS 点击 `读取状态` 后，把本地订单引用替换成真实物流单号，并把该预报重新标记为待推送。
4. 再点击 `写入预报`，WMS 会向 `预报` sheet 追加一行带真实物流单号的数据。
5. 后续对方按真实物流单号对接物流公司或仓库扫码，再回传 `DELIVERED`、`已收到`、异常等状态。

## 包裹预报状态回传匹配修正

- `apps/api/src/modules/package-prealerts/package-prealert-sheets-sync.service.ts`
  - `读取状态` 原来只按 `预报ID` 或 `物流单号` 匹配本地预报。
  - 现在增加 `链接`、`订单链接`、`Apple订单链接` 兜底匹配，可以用 Apple 订单原始链接或链接中的 Apple 订单号找到本地预报。
  - 如果本地当前还是 `APPLE-W...` 订单引用，而状态回传行带来了真实承运商 `物流单号`，系统会把本地预报单号替换成真实物流单号，并根据 `物流类型` 或 `物流查询链接` 识别 UPS/USPS/FedEx。
  - 匹配成功但状态行没有 `入库状态`、`入库日期`、`订单状态`、`提醒` 等业务状态时，只记录已读取，不会改变页面上的入库状态或预警数量。

- `docs/api/17-package-prealerts.md`, `docs/product/13-package-prealert-prd.md`
  - 补充状态回传的三层匹配规则：`预报ID` -> `物流单号` -> `链接/订单链接`。

### 使用逻辑

1. 对方或仓库在 `状态` sheet 回写时，最好带回 `预报ID`。
2. 如果没有 `预报ID`，可以带回真实 `物流单号`。
3. Apple 订单暂时没有真实物流单号时，可以带回原始 `链接`，WMS 会按 Apple 订单链接匹配。
4. 对方查到真实物流单号后，把它写到 `状态` sheet 的 `物流单号`；WMS 读取后会把页面里的 `APPLE-W...` 订单引用替换成真实物流单号。
5. 页面要出现变化，状态行还必须写入 `入库状态=已收到/已入库/RECEIVED`，或写入 `入库日期/入库时间`，或写入 `订单状态=DELIVERED`、`提醒=未收到` 等触发字段。

## 使用逻辑

1. 在 Google 表格中准备两个 sheet：`预报` 和 `状态`。
2. 两个 sheet 第一行必须是字段表头，字段名按 API 模板创建。
3. WMS API 环境变量配置 Google 服务账号和表格 ID。
4. 必须把 Google 表格共享给服务账号邮箱，并授予 Editor 权限。
5. WMS 创建包裹预报后，进入包裹预报页点击“写入预报”，系统把未推送数据追加到 `预报`。
6. 对方系统读取 `预报`，仓库扫码后写入 `状态`。
7. WMS 点击“读取状态”或“完整同步”，系统按 `预报ID` 或 `物流单号` 更新包裹状态。
8. 如果回传有入库日期，包裹变为已收到；如果回传显示 DELIVERED 但未收到，进入包裹预警。
9. 本地登录测试时，如果页面仍显示旧错误，刷新页面后重新输入密码即可；新版页面不会在未登录状态主动显示过期提示。
10. 本地管理员密码已临时重置后，可登录系统进入“系统设置 -> 账号安全”改成自己的密码。
