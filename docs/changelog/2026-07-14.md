# 2026-07-14

## Inbound hard stops, replenishment-note visibility, and outbound packing fixes

### What changed

- Inbound scanning now treats package tracking, UPC, and IMEI/Serial as distinct scan positions.
  A UPS or `BB0000` value scanned into IMEI/Serial, or the same value repeated across scan fields,
  stops auto-submit, flashes the incorrect field, and returns focus to that field.
- The API enforces the same cross-field rule before creating or updating an inbound preview row.
  Normal 15-digit IMEI values and Apple alphanumeric identifiers such as `SH9LRL91YFC` remain valid.
- Customer inventory rows now return and display force-inbound metadata. The `补货留言` column
  shows the reason entered during force inbound, for example `chen补给JH`, and inventory search also
  matches that reason.
- Deleting one item from an open outbound box now runs immediately after clicking `删除`; the extra
  browser confirmation was removed. Whole-box and batch-box deletion still require confirmation.
- The box detail editor no longer closes when the backdrop is clicked and no longer resets the box
  name because the box timestamp changed while the same modal is open. Operators can select all of
  the current name, replace it, and save through the explicit modal controls.
- Box-detail printing no longer repeats the full box name and its short box suffix. The first line
  is now `date + customer`, the next line is the short box label such as `箱2`, followed by product
  totals.

### Usage logic

1. Scan package tracking, then UPC, then IMEI. A value scanned into the wrong position stays visible,
   the field flashes red, and no next item is created until it is corrected.
2. When using force inbound for a `BB0000` replenishment row, enter the ownership explanation in the
   required reason prompt. Customer inventory displays that text under `补货留言`.
3. In an open outbound box, click a row's `删除` once to return it to customer inventory. Use the
   protected box-level delete flow only when deleting complete boxes.
4. In box detail, select and replace the full box name, then click `保存设置`. The modal stays open
   until an explicit close/cancel action.
5. `打印明细` shows the compact date/customer line and one box-label line before product totals.
