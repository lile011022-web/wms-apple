## 入库明细下载状态筛选

- `apps/web/src/pages/detail-download/page.tsx`
  - 明细下载页在 `入库明细` 报表类型下新增 `入库状态` 筛选栏，默认选择 `已确认入库`。
  - 操作员可切换为 `全部状态`、`待确认`、`异常`、`已作废`，用于排查问题或下载特定状态数据。
  - 预览摘要同步展示当前入库状态，生成导出前可确认下载范围，避免把待确认、异常或作废数据误认为正常入库数据。

- `docs/api/14-reports.md`
  - 补充 `filters.inboundStatus` 的接口用法，明确入库明细默认按 `CONFIRMED` 下载。

- `docs/product/10-report-rules.md`
  - 更新报表业务规则，说明入库明细导出需要按状态筛选，默认只下载已确认入库数据。

## 使用逻辑

- 日常客户对账、飞书登记和正式入库统计使用默认 `已确认入库`。
- 只有排查异常、核对草稿或追溯作废记录时，才切换到 `全部状态` 或指定的非确认状态。
- 后续如需调整状态枚举或默认策略，应同步维护前端筛选项、`ReportFilterDto.inboundStatus`、报表仓库查询和报表规则文档。

## 出库明细 Excel 客户核对表优化

- `apps/api/src/modules/reports/reports.service.ts`
  - 标准 `OUTBOUND_DETAIL` Excel 导出的 `出库信息` 工作表将 `销售客户` 改为 `归属客户`，将 `目的地` 改为 `实际送到地址`。
  - `实际送到地址` 只保留表格字段名，导出值固定为空，避免把箱子备注或客户联系信息误当成正式送达地址。
  - `SN&IMEI` 工作表新增 `单号` 列，展示每台物品的入库包裹/订单号 `inventoryItem.upsTrackingNo`，方便逐台核对来源单号。
  - 统一标准出库明细工作簿的单元格横向、纵向居中样式，保持客户交付表格格式一致。

- `apps/api/src/modules/reports/tests/reports.service.spec.ts`
  - 增加标准出库明细 Excel 的字段名、地址留空、单号列和单元格内容断言，防止后续改动退回旧模板。

- `docs/api/14-reports.md`
  - 同步标准出库明细 Excel 四个工作表的字段说明，明确 `归属客户`、`实际送到地址`、逐台 `单号` 和居中格式。

- `docs/product/10-report-rules.md`
  - 更新报表业务规则，说明装箱明细 Excel 的客户核对表固定包含送达地址占位字段和 SN&IMEI 单号列。

## 使用逻辑

- 操作员在明细下载页或出库装箱页导出 `OUTBOUND_DETAIL` + `Excel` 且未选择 `已装箱汇总表格` 时，会得到标准四工作表客户核对表。
- `出库信息` 用于整票信息核对，`SN&IMEI` 用于逐箱逐台核对上传单号、物品单号、UPC、SN 和 IMEI，`各箱型号汇总` 和 `出库详情` 继续提供型号数量汇总。
- 如果后续需要真正独立维护“实际送到地址”，应在出库箱或出库订单模型中新增专用字段，并把导出逻辑从当前固定空值切换到该字段。

## 使用逻辑

- 如果需要核对一个不知道属于哪个客户的物流单号，进入 `入库记录` 后保持客户为 `全部客户`，直接在搜索框输入单号查询即可。
- Dashboard 首页用于当天业务总览：四个大指标看总量，`今日入库客户明细` 看各客户当天分别入库多少。
- 出库装箱常用箱规优先从下拉框选择；不在四个常用尺寸里的箱子继续使用 `Custom`。
