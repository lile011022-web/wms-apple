# 2026-06-25

## 当前入库单相同物流单号提示

### 变化

- 入库物流单号检查现在同时统计当前草稿内已存在的相同物流单号。
- 操作员在当前入库单里再次扫描相同物流单号时，页面会显示“当前入库单已扫过该物流单号”并要求点击继续入库。
- 该提示不禁止同一包裹多件商品入库，只是防止未开启连续扫时误扫同一单号后静默通过。

### 修改位置

- `apps/api/src/modules/inbound/inbound.repository.ts`：新增当前草稿内物流单号计数。
- `apps/api/src/modules/inbound/inbound.service.ts`：`scanUps` 返回当前草稿重复状态和数量。
- `apps/web/src/pages/inbound-scan/page.tsx`：把当前草稿重复状态显示为物流单号异常提示。
- `docs/product/05-inbound-rules.md`、`docs/api/08-inbound-scan.md`：同步当前草稿重复提示规则。

### 使用逻辑

1. 操作员扫描物流单号。
2. 如果该单号已在当前入库草稿中出现，页面显示异常提示。
3. 操作员确认后可以继续加入明细；未确认时不能静默加入。

## 入库物流单号手动确认门禁

### 变化

- 收紧入库物流单号自动放行规则：只有 UPS 规则和 `9622` 开头的 22 位 FedEx 单号可以直接加入明细。
- USPS、非 `9622` FedEx、其它不符合规则的物流单号，以及已确认入库记录中重复出现的物流单号，都必须先显示异常提醒并由操作员点击继续入库。
- “同一物流单连续扫”现在不再只是保留输入值。启用该选项前会先检查当前物流单号，异常或重复时必须手动确认后才能开启连续扫。
- 加入明细前会再次检查物流单号，避免操作员跳过提示直接入库。

### 修改位置

- `packages/shared/src/validators/ups.ts`：新增 `isAutoAcceptedPackageTracking`，定义 UPS 和 `9622` FedEx 的自动放行规则。
- `apps/api/src/modules/inbound/inbound.service.ts`：扫描检查和保存明细都改为使用自动放行规则；异常物流单号必须带 `trackingExceptionConfirmed=true`。
- `apps/web/src/pages/inbound-scan/page.tsx`：连续扫开关和加入明细动作都接入物流单号复核，未确认异常时阻断继续操作。
- `docs/product/05-inbound-rules.md`、`docs/api/08-inbound-scan.md`：同步产品规则和 API 约束。

### 使用逻辑

1. 操作员扫描物流单号。
2. 如果是 UPS 或 `9622` FedEx，页面允许继续扫描 UPC/IMEI 并加入明细。
3. 如果是重复单号、USPS、非 `9622` FedEx 或其它异常格式，页面显示原因，操作员必须点击继续入库。
4. 确认后当前物流单号可以加入明细；未确认时不能开启同一物流单连续扫，也不能加入明细。

## 客户库存 IMEI 明细搜索

### 变化

- 在客户库存页的 `IMEI 明细` 分页区域增加搜索框，可按单号、IMEI、Serial、UPC、SKU 或商品名称筛选明细行。
- 搜索条件会传给 `GET /inventory/items` 的 `search` 参数，并保留当前客户、仓库和分页大小；输入变化后自动回到第 1 页。

### 修改位置

- `apps/web/src/pages/customer-inventory/page.tsx`：新增明细搜索状态、查询参数和搜索输入框。
- `apps/web/src/components/pagination-controls.tsx`：支持在分页范围和分页按钮之间插入页面专用控件。
- `apps/web/src/styles/globals.css`：新增客户库存明细工具条布局。
- `docs/product/06-inventory-rules.md`、`docs/api/10-inventory.md`：同步客户库存明细搜索规则。

### 使用逻辑

1. 进入客户库存页并选择客户/仓库。
2. 在 `IMEI 明细` 上方搜索框输入物流单号、IMEI、UPC、SKU 或商品关键词。
3. 系统按当前客户和仓库重新读取匹配明细，分页显示筛选后的库存行。

## 出库批量装箱筛选和分箱预览

### 变化

- 出库装箱的客户库存表新增 `入库时间` 列，直接显示库存 `receivedAt`，方便按入库先后挑选装箱货物。
- 在批量装箱模式的可装箱货物表头增加 `成色` 和 `品类` 筛选，可组合筛出全新/翻新与 iPhone/iPad；选择 `翻新 + 全部品类` 可把翻新手机和平板一起作为装箱候选。
- 细致装箱模式不显示也不应用成色/品类筛选，继续只依赖扫码匹配和普通搜索。
- 批量装箱的成色/品类筛选改为按当前客户和仓库的全量可装箱库存计算，不再只按当前可见页 10 条计算；分箱按钮数量、分页总数和批量弹窗数据源保持一致。
- 可装箱货物勾选现在会在同一客户和仓库下跨搜索、筛选、分页和分页大小保持记忆；切换客户或仓库时清空，避免跨客户/跨仓库混装。
- 在批量装箱操作区增加 `取消勾选` 按钮，可一次清空跨页记忆的已选货物。
- 批量分箱弹窗优先使用已勾选货物；没有勾选时使用当前筛选后的可装箱库存。
- 填写每箱数量时，弹窗即时显示每个箱子的计划明细，包括入库时间、商品名称、IMEI/Serial 或物流单号。
- 箱子明细弹窗新增 `打印明细` 按钮，点击后先弹出文本式打印预览；标题使用日期、客户名称和箱子名称，明细按商品名称汇总为 `商品名称*数量` 并显示 `Total`，再由 `确认打印` 打开浏览器打印。

### 修改位置

- `apps/web/src/pages/outbound-packing/page.tsx`：新增成色/品类筛选状态、全量筛选计数、勾选记忆、取消勾选动作、入库时间列、批量分箱数据源、每箱明细预览和箱子明细打印预览。
- `apps/web/src/styles/globals.css`：新增出库筛选控件、批量分箱预览和打印预览/打印专用布局。
- `docs/product/07-outbound-rules.md`：同步出库批量装箱筛选、细致装箱不显示筛选、勾选记忆、分箱预览和打印明细规则。
- `docs/api/11-outbound-packing.md`：说明可装箱返回行需要保留 `receivedAt` 和产品/UPC 文本以支撑前端筛选。

### 使用逻辑

1. 进入出库装箱，选择客户和仓库。
2. 切换到批量装箱模式，在客户库存表中按 `成色` 和 `品类` 筛选，例如 `翻新 + 全部品类`。
3. 手动勾选需要装箱的货物；切换筛选或分页后，已勾选数量会继续保留。
4. 点击 `分箱装箱`，输入箱数和每箱数量。
5. 核对弹窗里的每箱计划明细，数量合计等于已勾选或筛选库存总数后确认批量装箱。
6. 在箱子明细中点击 `打印明细`，确认预览内容后点击 `确认打印`。

## 客户管理行内编辑

### 变化

- 客户管理列表新增 `操作` 列，每个客户行提供 `编辑` 和 `保存` 按钮。
- 点击 `编辑` 后，当前行的客户编码和客户名称会切换为输入框，可直接修改。
- 点击 `保存` 后调用 `PATCH /customers/:id` 保存客户编码和名称，并刷新客户列表及全局客户下拉选项。
- 保存前会校验客户编码和客户名称不能为空，保存失败时页面显示错误提示。

### 修改位置

- `apps/web/src/pages/customer-management/page.tsx`：新增行内编辑状态、保存 mutation、错误提示和操作列。
- `apps/web/src/styles/globals.css`：新增客户行操作按钮布局。
- `docs/product/03-customer-rules.md`：同步客户管理行内编辑规则。

### 使用逻辑

1. 进入客户管理页。
2. 在客户列表中点击目标客户行的 `编辑`。
3. 直接修改客户编码或客户名称。
4. 点击 `保存`，系统保存后刷新列表和其它流程使用的客户选项。

## 装箱明细导出上传单号

### 变化

- 装箱明细报表新增 `shippingTrackingNo` 导出字段，对应出库装箱卡片里的 `上传单号` / 出库运单号。
- 普通预览和 CSV/常规字段导出可以选择 `shippingTrackingNo`，用于按行查看每条装箱明细所属箱子的上传单号。
- `OUTBOUND_DETAIL + EXCEL` 的客户核对样表在每个箱子的明细区域显示 `上传单号：...`，未填写时显示 `-`。

### 修改位置

- `apps/api/src/modules/reports/reports.service.ts`：把 `outboundBox.shippingTrackingNo` 加入装箱明细字段白名单、预览/CSV 读取逻辑和固定 Excel 样表。
- `apps/api/src/modules/reports/tests/reports.service.spec.ts`：覆盖预览字段和 Excel 样表中的上传单号输出。
- `docs/api/14-reports.md`、`docs/product/10-report-rules.md`：同步装箱明细导出字段和 Excel 样表规则。

### 使用逻辑

1. 出库装箱卡片点击 `上传单号` 并保存出库运单号。
2. 进入明细下载，选择 `装箱明细`。
3. 如果导出 CSV 或字段表格，选择 `shippingTrackingNo` 字段。
4. 如果导出 Excel 样表，每个箱子标题区域会自动带出对应上传单号。
