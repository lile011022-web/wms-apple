# 2026-06-29

## 装箱明细新增已装箱汇总表格模式

- 前端 `apps/web/src/pages/detail-download/page.tsx`：当报表类型为 `装箱明细` 且格式为 `Excel` 时，新增 `表格模式` 下拉框；默认仍是 `现有客户核对表`，可切换到 `已装箱汇总表格`。
- 前端 `apps/web/src/pages/detail-download/page.tsx`：当报表类型为 `库存明细` 且格式为 `Excel` 时，也新增 `留仓汇总表格` 模式；选择后自动按 `IN_STOCK` 留仓库存导出。
- 后端 `apps/api/src/modules/reports`：新增 `exportLayout` 导出参数和 `PACKED_SUMMARY`、`WAREHOUSE_HOLD` 布局。`OUTBOUND_DETAIL + EXCEL + STANDARD` 保持原四个工作表，`PACKED_SUMMARY` 输出一个 `已装箱汇总` 工作表，`INVENTORY_DETAIL + EXCEL + WAREHOUSE_HOLD` 输出一个 `留仓汇总` 工作表。
- 新汇总表格的使用逻辑：左侧按每台机器列出 `箱数 / upc / 型号 / imei`，右侧按 `箱号 + UPC + 型号` 汇总数量，底部绿色 `总数` 行显示全部已装箱明细数量。
- 留仓汇总表格的使用逻辑：系统库存没有实体留仓箱号时，按导出结果每 24 台自动生成 `留仓箱1`、`留仓箱2` 等虚拟箱号，最后不足 24 台的保留为最后一箱。
- 相关维护位置：导出布局生成在 `apps/api/src/modules/reports/reports.service.ts`；前端表格模式入口在明细下载页；产品规则见 `docs/product/10-report-rules.md`，API 合同见 `docs/api/14-reports.md`。
