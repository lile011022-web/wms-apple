## Module Purpose

- `apps/api/src/modules/auth`: owns login, logout, refresh token, `/auth/me`, token issuing, disabled-user login blocking, and login/logout audit records.
- `apps/api/src/modules/users`: owns system user CRUD, password hashing, user status changes, role assignments, and user-change audit records.
- `apps/api/src/modules/roles`: owns role listing and permission assignment replacement for authorization management.
- `apps/api/src/modules/permissions`: owns permission point listing for the role authorization UI.
- `apps/api/src/modules/audit-logs`: owns creation of critical operation audit log rows.
- `apps/api/src/common/guards`: owns reusable JWT authentication and permission enforcement.
- `apps/api/src/common/decorators`: owns reusable `@CurrentUser()` and `@Permissions()` endpoint metadata.
- `README.md`: gives contributors the current project baseline, directory ownership, and next development sequence.
- `docs/api/03-auth.md`: documents authentication request/response contracts and token behavior.
- `docs/api/04-users-roles-permissions.md`: documents user, role, and permission management contracts.

## Usage Logic

After local PostgreSQL is migrated and seeded, log in with the seeded development admin:

```bash
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "content-type: application/json" \
  -d '{"email":"admin@wms-scan.local","password":"<SEED_ADMIN_PASSWORD>"}'
```

Use the returned access token for protected endpoints:

```bash
curl http://localhost:3000/api/v1/auth/me \
  -H "authorization: Bearer <accessToken>"
```

System Settings user management can call:

```text
GET /api/v1/users
POST /api/v1/users
PATCH /api/v1/users/:id
GET /api/v1/roles
PATCH /api/v1/roles/:id/permissions
GET /api/v1/permissions
```

User and role write endpoints create audit logs. Password hashes are never returned in API responses or audit snapshots.
