## Module Purpose

- `packages/shared`: owns business enums and reusable scan validation rules that must stay consistent between frontend and backend.
- `apps/api`: owns NestJS API startup, global HTTP behavior, health check, Swagger documentation, and future backend modules.
- `apps/web`: owns the React application shell, routing, layout, and future route-level pages.
- `docker-compose.yml`: local PostgreSQL and Redis services for later backend development.
- `.env.example`: documents local environment variables without creating or committing a real `.env`.

## Usage Logic

Install dependencies:

```bash
pnpm install
```

Start the frontend:

```bash
pnpm dev:web
```

Start the API:

```bash
pnpm dev:api
```

Check the API health endpoint:

```bash
curl http://localhost:3000/api/v1/health
```

Open Swagger:

```text
http://localhost:3000/api/docs
```

Run project checks:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```
