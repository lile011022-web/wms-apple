## Module Purpose

- `apps/api/src/common/errors`: owns stable backend error codes and expected business-rule exception objects.
- `apps/api/src/common/filters`: converts framework, validation, business, and unexpected exceptions into the shared API failure envelope.
- `apps/api/src/common/dto`: owns reusable query DTOs such as pagination, search, and sort defaults for list endpoints.
- `apps/api/src/main.ts`: owns global HTTP behavior, Swagger groups, and global Bearer Auth documentation.
- `apps/api/src/health`: owns the bootstrapped health-check endpoint and its Swagger tag.
- `apps/api/package.json`: owns API-local Prisma scripts that must point at `apps/api/prisma/schema.prisma`.
- `docs/api`: documents API response shape, request ID behavior, pagination defaults, and error codes for frontend integration.
- `docs/database`: documents the local database service direction and Prisma ownership boundary before full schema modeling starts.
- `docs/architecture/backend-implementation-roadmap.md`: explains the backend module titles, implementation order, API/documentation ownership, testing focus, and deployment path.
- `docs/architecture/README.md`: lists the available architecture documents so future maintainers can find the roadmap from the folder entrypoint.
- `packages/shared`: owns business enums and reusable scan validation rules that must stay consistent between frontend and backend.
- `apps/api`: owns NestJS API startup, global HTTP behavior, health check, Swagger documentation, and future backend modules.
- `apps/web`: owns the React application shell, routing, layout, and future route-level pages.
- `docker-compose.yml`: local PostgreSQL and Redis services for later backend development.
- `.env.example`: documents local environment variables without creating or committing a real `.env`.

## Usage Logic

Use the backend roadmap as the main planning document before starting backend business modules:

```text
docs/architecture/backend-implementation-roadmap.md
```

Install dependencies:

```bash
pnpm install
```

Start the frontend:

```bash
pnpm dev:web
```

Start the API:

```bash
pnpm dev:api
```

Check the API health endpoint:

```bash
curl http://localhost:3000/api/v1/health
```

Open Swagger:

```text
http://localhost:3000/api/docs
```

Run project checks:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Use `BusinessError` in services for expected WMS rule failures, and reuse `PaginationQueryDto` from `apps/api/src/common/dto` for future list endpoints.

From `apps/api`, validate Prisma schema and generate Prisma Client with:

```bash
pnpm prisma:validate
pnpm prisma:generate
```

From the repository root, start local PostgreSQL and Redis with:

```bash
docker compose up -d postgres redis
```
