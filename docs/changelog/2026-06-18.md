## Touched Modules And Purpose

- `apps/web/Dockerfile`: builds the Vite frontend with explicit pnpm 9.15.0 Corepack activation, `VITE_API_BASE_URL=/api/v1` by default, and serves the static `dist` output through nginx.
- `apps/api/Dockerfile`: builds shared code with explicit pnpm 9.15.0 Corepack activation, generates Prisma Client, builds the NestJS API, and starts `dist/main.js` on internal port 3000.
- `.dockerignore`: keeps local dependencies, real environment files, build output, backups, and cache folders out of Docker image build contexts.
- `docker-compose.prod.yml`: defines the internal-test production topology with `web`, `api`, `postgres`, and `redis` services plus persistent database and Redis volumes.
- `nginx/default.conf`: serves the frontend at `/`, forwards `/api` to `api:3000`, supports React Router refreshes, and allows uploads up to 20 MB.
- `.env.production.example`: documents production environment variables with placeholder-only values and no real credentials.
- `.gitignore`: prevents accidental commits of the real `.env.production` file used on the VPS and generated PostgreSQL backup archives.
- `infra/scripts/deploy.sh`: runs the repeatable VPS update flow with an explicit Docker Compose project name, skips `git pull` when `.git` is absent, then builds, starts, reports status, and prints access hints.
- `infra/scripts/backup-postgres.sh`: creates timestamped compressed `pg_dump` backups under `backups/`, uses an explicit Docker Compose project name, and removes backups older than the retention window.
- `docs/deployment`: owns low-cost VPS deployment, rollback, DNS, Cloudflare Access, backup, and internal-test operating guidance.
- `.github/workflows/deploy.yml`: is a disabled-by-default manual placeholder for future SSH-based VPS deployment secrets.
- `apps/api/src/modules/reports/dashboard`: owns Dashboard metric endpoints and source queries.
- `apps/api/src/modules/audit-logs`: owns audit-log writing, recent operation logs, and paginated audit-log lookup.
- `apps/api/prisma/seed.ts`: keeps local admin role aligned with new backend permissions.
- `apps/api/package.json` and `apps/api/jest.e2e.config.cjs`: define repeatable API unit/e2e test commands.
- `apps/api/src/modules/core-workflows/tests`: owns cross-module regression protection for the warehouse business flow.
- `apps/api/src/app.module.ts`: owns application-wide module registration, including globally available JWT support for route guards.
- `apps/api/src/modules/inventory` and `apps/api/src/modules/reports`: own inventory/report repositories and must import `DatabaseModule` so Prisma can be injected at runtime.
- `packages/shared`: owns shared enums, validators, and types, and now emits Node-compatible ESM dist files for API production startup.
- Formatting-only API files keep existing behavior unchanged while allowing full-repo Prettier checks to pass.
- `apps/web/src/layouts`: owns local login/logout and current-user state so protected pages can actually be used in the browser.
- `apps/web/src/pages`: now owns API-backed operational pages for Dashboard, inbound scan, inbound records, customer inventory, outbound packing, exception handling, batch customer change, detail download, UPC catalog, customer management, and system settings.
- `apps/api/src/modules/outbound`: owns customer-linked outbound box number generation, box listing, packing, and sealing rules.
- `apps/api/src/modules/reports`: owns report preview row sampling, export file generation, export history, and download payloads.
- `apps/web/src/pages/detail-download`: owns report preview, export creation, export history, and browser download triggering for generated files.
- `apps/web/src/pages/upc-library`: owns UPC manual creation, UPC lookup, import-template download, CSV parsing, upload preview, and bulk import submission.
- `docs/operations`: owns clone-to-run operational guidance for new machines.
- `docs/architecture/git-branching.md`: owns branch naming and promotion rules for `main`, `develop`, and short-lived work branches.
- `docs/api`: describes request and response contracts plus test command entry points.

## Usage Logic

Low-cost internal-test deployment uses:

```text
Browser -> nginx web container :80
nginx / -> React static files
nginx /api -> api:3000
api -> postgres:5432 over Docker network
api -> redis:6379 over Docker network
```

Local production-shape verification:

```text
cp .env.production.example .env.production
docker compose -f docker-compose.prod.yml --env-file .env.production build
docker compose -f docker-compose.prod.yml --env-file .env.production up -d
docker ps
http://localhost/
http://localhost/api/v1/health
```

VPS update flow:

```text
PROJECT_DIR=/opt/wms-scan infra/scripts/deploy.sh
```

PostgreSQL backup flow:

```text
PROJECT_DIR=/opt/wms-scan infra/scripts/backup-postgres.sh
```

The committed files intentionally contain only placeholders. The real `.env.production`, real VPS host, real SSH key, real domain, and real database password must be created outside Git.

Dashboard pages call:

```text
GET /api/v1/dashboard/summary
GET /api/v1/dashboard/trends
GET /api/v1/dashboard/exception-distribution
GET /api/v1/dashboard/top-inbound-customers
```

Each Dashboard endpoint accepts optional `warehouseId` and requires `dashboard.read`.

Audit-log pages call:

```text
GET /api/v1/audit-logs/recent
GET /api/v1/audit-logs
```

`GET /audit-logs` supports pagination, search, action/resource/operator/request/date filters, and requires `audit-logs.read`.

Frontend pages use `request<T>()` from `apps/web/src/api/client.ts` so all successful responses are unwrapped from the backend API envelope. When a user has tokens in local storage, the client sends the access token as a Bearer token.

The System Settings page calls:

```text
GET /api/v1/warehouses?isActive=true
GET /api/v1/settings
PATCH /api/v1/settings
```

Operators edit default warehouse, scan rules, exception creation rules, notification switches, and retention days in one page. Saving sends the grouped settings payload back to `PATCH /settings`; the backend still owns permission checks, validation ranges, audit logging, and default warehouse existence checks.

Test commands:

```text
pnpm --filter @wms-scan/api test:unit
pnpm --filter @wms-scan/api test:e2e
pnpm --filter @wms-scan/web test
pnpm --filter @wms-scan/shared test
pnpm test
```

Inbound Scan usage after the interaction fix:

```text
Open /inbound-scan.
UPS, UPC, and IMEI start empty.
Lock a customer and warehouse to create a draft.

Partially typed UPS, UPC, and IMEI values are kept when switching between frontend pages, but reset after refreshing the browser.
```
