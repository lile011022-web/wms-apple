### Usage Logic

1. 默认启动系统时，不会出现“包裹预报”和“包裹预警”菜单。
2. 入库扫码页面不会根据预报单号自动带出客户，操作员按原流程手动选择客户并锁定。
3. 确认入库只执行库存入库、异常和审计逻辑，不再改动预报记录。
4. 如果后续决定继续做包裹预报，在 API 和 Web 环境中同时打开两个开关后重启服务即可恢复入口与联动。

## Inbound Scan Strict Validation

- `apps/api/src/modules/inbound/inbound.service.ts`
  - 收紧扫码入库新增明细规则：商品库不存在的 UPC 直接拒绝，不再追加异常明细。
  - 已存在库存的 IMEI/Serial 直接拒绝新增明细，不再先追加异常明细。
  - `trackingExceptionConfirmed` 只允许 USPS、非 9622 FedEx 等合法但需人工确认的物流单号；随机文本或不符合任何物流规则的值仍然拒绝。

- `apps/web/src/pages/inbound-scan/page.tsx`
  - 在扫码页面增加本地格式预校验，物流单号、UPC、IMEI 不符合规则时直接显示中文原因并禁用加入明细。
  - 优先显示后端字段级校验详情，避免只显示 `Validation failed`。

- `apps/api/src/modules/inbound/tests/inbound.service.spec.ts`
  - 补充并更新扫码规则矩阵：UPS、BB0000、9622 FedEx 自动放行；USPS/非 9622 FedEx 需要确认；随机物流单号、未维护 UPC、重复库存 IMEI 直接拒绝。

- `docs/product/05-inbound-rules.md`, `docs/api/08-inbound-scan.md`
  - 同步新的扫码阶段规则：错误数据不应先进入当前入库单明细，操作员需要先修正再继续。

### Usage Logic

1. 一版模式必须具备合法物流单号、商品库已维护 UPC、合法 IMEI 后才能加入明细。
2. 物流+UPC 模式允许不扫 IMEI，但 UPC 仍必须先在商品库中存在。
3. USPS 和非 9622 FedEx 属于合法但需确认的物流单号；确认后可以加入明细。
4. 随机数字、随机文本、未维护 UPC、库存重复 IMEI/Serial 都会在扫码阶段被拒绝。

## Inbound Draft Recovery And Review Editing

- `apps/api/src/modules/inbound/inbound.controller.ts`, `apps/api/src/modules/inbound/inbound.service.ts`, `apps/api/src/modules/inbound/inbound.repository.ts`
  - 新增 `GET /api/v1/inbound/drafts/latest/my`，用于恢复当前登录操作人自己的最新未完成入库草稿。
  - 恢复范围只包含当前操作人的 `DRAFT` 入库单，不返回已确认入库单或其他操作人的草稿。

- `apps/web/src/api/workflow.ts`, `apps/web/src/pages/inbound-scan/page.tsx`
  - 入库扫码页面打开时，如果本地没有有效草稿锁定，会自动恢复当前操作人最新未完成入库单，并同步客户、子客户、仓库和待入库明细。
  - 将“当前入库单”整块移动到客户锁定区域下方、扫码模式和扫码输入区域上方。
  - 待入库明细表支持确认入库前编辑、保存、取消和删除；保存时继续调用后端重跑入库校验规则。

- `apps/web/src/styles/globals.css`
  - 为当前入库单表格增加横向滚动容器、稳定列宽和固定操作列，避免编辑/保存/删除按钮在页面右侧不可见。

- `apps/api/src/modules/inbound/tests/inbound.service.spec.ts`
  - 增加当前操作人最新草稿恢复的服务测试。

- `docs/product/05-inbound-rules.md`, `docs/api/08-inbound-scan.md`
  - 同步自动恢复草稿、确认前复核编辑、操作列可见性和待入库明细删除规则。

### Usage Logic

1. 操作员离开入库扫码页面再回来时，系统优先接回该操作员自己的最新未完成入库草稿。
2. 选择客户并锁定后，当前入库单先展示在页面上方，操作员可以先复核已有明细再继续扫码。
3. 待入库明细在确认入库前可以编辑、保存、取消或删除；保存不会新增第二行，而是覆盖原行并重新校验。
4. 明细操作按钮保持在当前入库单表格右侧可见，窄屏时通过横向滚动查看数据列。

- `apps/web/src/pages/outbound-packing/page.tsx`
  - 新建或打开箱子后的箱子明细弹窗增加箱子设置编辑区，可修改箱子名称、尺寸、重量、出库单号 / 面单号和备注。
  - 弹窗保存后同步当前箱子、箱子列表和弹窗汇总；清空备注或单号后保存也会真正清空。
  - 手动调整尺寸时按 `CUSTOM` 保存，避免手动长宽高被固定预设覆盖。
  - 装箱页面滚动后显示“回到顶部”按钮，点击后平滑回到页面顶部。

- `apps/web/src/styles/globals.css`
  - 为箱子明细弹窗编辑区和回到顶部按钮补充响应式布局，避免按钮遮挡表格操作。

- `apps/api/src/modules/outbound/tests/outbound.service.spec.ts`
  - 更新出库箱子编辑测试，覆盖新的常用尺寸预设。

- `docs/product/07-outbound-rules.md`, `docs/api/11-outbound-packing.md`
  - 同步箱子明细弹窗可编辑、尺寸预设白名单和回到顶部交互规则。

### Usage Logic

1. 新建箱子后弹出的明细中可以直接修正箱子基础信息，不需要回到顶部快速编辑区。
2. 箱子未封箱时可保存名称、尺寸、重量、出库单号和备注；已封箱箱子仍需先返工再编辑。
3. 固定尺寸直接按预设保存，打开手动调整后按自定义尺寸保存。
4. 装箱页面下滚后可一键回到顶部，方便继续选择客户、创建箱子或扫码装箱。
