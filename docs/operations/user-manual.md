# WMS Scan 系统操作手册

适用版本：2026-07-09 生产版本
适用对象：仓库入库员、仓库出库员、客服对账人员、主管和系统管理员

这份手册用于让新人按实际业务顺序上手 WMS Scan。日常操作时优先按页面菜单走；遇到异常时，先确认客户、仓库、UPC、IMEI/Serial、库存状态和箱子状态。

## 1. 系统做什么

WMS Scan 用于美国仓库 Apple 产品的入库扫码、库存查询、出库装箱、异常处理、客户归属修正、明细下载和操作审计。

完整业务链路：

1. 先维护客户、商品 UPC、仓库和系统设置。
2. 入库时选择客户和仓库，锁定后扫描物流单号、UPC、IMEI 或 Serial。
3. 确认入库后，系统生成客户名下库存，状态为 `IN_STOCK`。
4. 出库时选择同一个客户和仓库，只能装入 `IN_STOCK` 库存。
5. 货物加入箱子后状态变为 `PACKED`。
6. 封箱前必须上传照片或视频证据。
7. 客户归属修正、异常处理、入库确认、封箱和报表导出都会留下审计记录。

## 2. 新人最快上手路线

1. 登录系统，确认左侧菜单和当前账号。
2. 到 `客户管理` 熟悉客户编码和客户名称。
3. 到 `商品管理` 熟悉 SKU、UPC、商品名称、型号和 IMEI 要求。
4. 到 `入库扫码` 做一单测试：锁定客户、扫码、复核、确认入库。
5. 到 `客户库存` 搜索刚入库的 IMEI、Serial 或物流单号。
6. 到 `出库装箱` 创建箱子，扫码或批量装箱，上传证据，封箱。
7. 到 `明细下载` 按入库批次、库存或箱子导出表格。
8. 到 `入库记录`、`异常池`、`批量修改客户` 学习错误数据的修正路径。

## 3. 登录和账号

登录用于识别当前操作人。所有关键操作都会记录操作人、时间、动作和修改内容。

操作步骤：

1. 打开系统地址。
2. 输入邮箱和密码。
3. 点击 `登录`。
4. 登录后左侧显示功能菜单。
5. 页面数据不是最新时，点击页面上的刷新或重新查询。
6. 离开岗位前退出账号。

注意事项：

- 不要多人共用一个账号，否则后续无法追责。
- 账号停用后不能登录。
- 登录过期后需要重新登录。

## 4. Dashboard

`Dashboard` 是运营看板，用来查看当天入库、封箱、库存、异常和最近操作。

查看重点：

- 今日入库数量。
- 今日封箱数量。
- 当前在库数量。
- 待处理异常数量。
- 最近 7 天入库、出库、异常趋势。
- 最近审计操作记录。

使用方法：

1. 点击左侧 `Dashboard`。
2. 查看今日数据和异常数量。
3. 异常数量不为 0 时，进入 `异常池` 处理。
4. 需要追踪近期操作时，查看最近审计记录或到 `明细下载` 导出审计日志。

## 5. 客户管理

客户是库存归属的基础。入库、库存、出库、报表和客户变更都围绕客户展开。

常用操作：

- 新增客户。
- 编辑客户编码和客户名称。
- 停用不再使用的客户。
- 查询客户相关业务数据。

操作步骤：

1. 点击左侧 `客户管理`。
2. 新增客户时填写 `客户编码` 和 `客户名称`。
3. 点击新增或保存。
4. 修改客户时，在列表中点击编辑。

注意事项：

- 客户编码是稳定业务标识，系统会转成大写。
- 不要随意删除客户，历史入库和库存需要保留客户归属。
- 入库时选错客户，确认入库后应走 `批量修改客户`，不要在出库时临时改客户。

## 6. 商品管理

`商品管理` 维护 UPC 商品库。入库扫码时，UPC 必须匹配启用商品，系统才允许正常加入待入库明细。

常用操作：

- 新增商品和 UPC。
- 查询 UPC 对应的商品。
- 下载商品导入模板。
- 批量导入商品。

单个新增：

1. 点击左侧 `商品管理`。
2. 填写 `SKU`、商品名称、型号、颜色、容量和 UPC。
3. 确认是否要求 IMEI。
4. 点击新增或保存。

批量导入：

1. 点击 `下载导入模板`。
2. 按模板填写商品资料。
3. 一个商品多个 UPC 时，用英文分号 `;` 分隔。
4. 上传文件并确认导入。

注意事项：

- UPC 必须是数字，并且全系统唯一。
- SKU 必须唯一，系统会转成大写。
- 入库扫描只能使用启用状态的商品和 UPC。
- 商品要求 IMEI 时，确认入库前必须提供合法 IMEI 或系统认可的 Apple 设备标识。

## 7. 系统设置

`系统设置` 用于维护默认仓库、扫码规则、异常规则、审计和数据保留策略。

常见设置：

- 默认仓库。
- 入库是否必须锁定客户。
- 出库是否强制客户归属一致。
- 是否检查重复 IMEI/Serial。
- 是否检查重复物流单号。
- 是否自动生成异常。
- 审计、报表、异常数据保留天数。

注意事项：

- 系统设置会影响后续所有入库、出库和异常处理。
- 普通操作员不要随意修改系统设置。
- 设置修改会写入审计日志。

## 8. 入库扫码

`入库扫码` 是收货核心页面。操作员先选择并锁定客户，再扫描物流单号、UPC、IMEI 或 Serial，最后复核并点击 `确认入库`。

### 8.1 页面顺序

当前页面从上到下的操作区：

1. 批量入库文件导入。
2. 客户、子客户 / 别名、仓库选择和客户锁定。
3. 当前入库单，也就是确认入库前的复核区。
4. 扫码模式和扫码输入区。
5. 待入库明细、UPC 核查和分页列表。

`当前入库单` 会显示在客户锁定区域下面、扫码输入区域上面，方便操作员先复核已有草稿再继续扫码。

### 8.2 客户锁定

操作步骤：

1. 点击左侧 `入库扫码`。
2. 选择客户。
3. 如有需要，选择子客户 / 别名。
4. 选择仓库。
5. 点击 `锁定客户`。

规则：

- 入库草稿创建后不能更换客户。
- 子客户 / 别名只记录收货来源，库存归属仍是父客户。
- 客户选错且已经确认入库后，走 `批量修改客户`。

### 8.3 最新草稿自动恢复

如果操作员扫码过程中临时打开别的页面，再回到 `入库扫码`：

- 系统会自动恢复当前登录操作人自己的最新未完成 `DRAFT` 入库单。
- 会同步恢复客户、子客户 / 别名、仓库、汇总和待入库明细。
- 不会恢复其他操作人的草稿。
- 不会恢复已经确认入库的批次。

如果自动恢复的不是你要继续处理的批次，可以在 `按入库单号恢复` 输入正确的 `INB-...` 批次号恢复。

### 8.4 扫码模式

入库支持两种模式：

- `一版模式`：物流单号、UPC、IMEI 都齐全后自动加入当前入库单。
- `物流+UPC 模式`：物流单号和 UPC 齐全后自动加入当前入库单，适合暂不录 IMEI 的场景；最终确认仍受商品规则限制。

手动 `加入明细` 按钮保留为补救入口。扫码仪没有正确触发自动加入时，可以检查三项输入后手动加入。

### 8.5 物流单号规则

自动放行：

- UPS 常规单号。
- 以 `9622` 开头、总长度 22 到 34 位的 FedEx 单号。
- 完整的 34 位纯数字物流条码，例如 `1119212621960001972000533804475274`。
- 以 `BB0000` 开头的仓库赔偿包裹号，`BB0000` 本身也允许。

需要人工确认后才可继续：

- USPS。
- 非 `9622` 开头但属于合法格式的 FedEx。
- 历史已确认记录中出现过的重复物流单号。
- 当前草稿中已经出现过但业务上确实属于同一包裹多件货的物流单号。

直接拒绝：

- 随机数字。
- 随机文本。
- 不符合 UPS、USPS、FedEx 或 `BB0000` 规则的值。

### 8.6 UPC、IMEI 和 Serial 规则

UPC：

- 必须是数字。
- 必须已在 `商品管理` 中维护并启用。
- 未维护 UPC 会在扫码阶段直接拒绝，不会加入待入库明细。
- UPC 缺失或格式错误时，无论扫码枪发送 Enter/Tab、页面自动跳转，还是人工点击 IMEI
  输入框，系统都会锁回 UPC；修正前不能继续扫 IMEI。物流单号异常时同样不能进入 UPC。

IMEI / Serial：

- 商品要求 IMEI 时，必须提供合法 IMEI 或系统认可的 Apple 设备标识。
- 一个明细不能同时填写 IMEI 和 Serial。
- 当前草稿内重复 IMEI/Serial 会被拒绝。
- 历史库存中已存在的 IMEI/Serial 会被拒绝。
- 如果把 UPS 或 `BB0000` 物流单号扫到 IMEI/Serial 框，系统会立即闪烁标记并锁回
  该输入框，修正前不会加入下一条。
- 物流单号、UPC、IMEI/Serial 三个位置不能重复使用同一个值。

### 8.7 待入库明细编辑

确认入库前，待入库明细可以直接处理：

- `编辑`：修改物流单号、UPC、IMEI 或 Serial。
- `保存`：覆盖原行，并重新执行扫码入库规则。
- `取消`：放弃本次编辑。
- `删除`：从当前草稿中移除该行。

规则：

- 保存编辑不会新增第二行，而是覆盖原来的待入库行。
- 保存后仍然会检查物流单号、UPC、IMEI/Serial 和重复值。
- 异常行必须先编辑修正或删除，才能继续扫码或确认入库。
- 表格右侧操作按钮会保持可见；屏幕较窄时，可横向滚动查看更多数据列。

### 8.8 标准入库流程

1. 选择客户、子客户 / 别名和仓库。
2. 点击 `锁定客户`。
3. 选择扫码模式。
4. 扫描或输入物流单号。
5. 扫描或输入 UPC。
6. 按商品要求扫描或输入 IMEI / Serial。
7. 系统自动加入明细，或点击 `加入明细`。
8. 在 `当前入库单` 检查产品件数、UPC 种类、商品款数、物流单号、待确认和异常数量。
9. 检查待入库明细。
10. 确认无误后点击 `确认入库`。
11. 系统生成库存，库存状态为 `IN_STOCK`。

### 8.9 批量文件导入

1. 在 `入库扫码` 下载入库模板。
2. 按模板填写 `单号`、`upc`、`imei`。
3. 上传 CSV 文件。
4. 系统把有效行加入当前草稿。
5. 逐行查看失败原因，修正文件或手动补录。
6. 复核当前入库单。
7. 点击 `确认入库`。

注意：文件导入只会加入草稿，不会自动确认入库。

### 8.10 常见入库问题

UPC 不识别：

1. 到 `商品管理` 查询 UPC。
2. 如果没有维护，先新增或导入商品 UPC。
3. 回到 `入库扫码` 重新扫描。

IMEI 重复：

1. 到 `客户库存` 或 `入库记录` 搜索 IMEI。
2. 确认是否已经入库。
3. 如果确实扫错，在待入库明细中编辑或删除。

物流单号异常：

1. 检查是否属于 UPS、USPS、FedEx 或 `BB0000`。
2. 合法但需要人工确认的单号，确认后继续。
3. 随机值或错误值必须改正，不能强行加入。

## 9. 入库记录

`入库记录` 用于查询已确认入库或异常入库明细，并处理确认后的错误。

常用操作：

- 按客户、状态、单号、UPC、IMEI、Serial 查询。
- 修正确认后的物流单号、UPC、IMEI 或 Serial。
- 对确认后仍为异常且经人工复核可以入库的记录执行强制入库。

修正入库记录：

1. 点击左侧 `入库记录`。
2. 搜索要修正的记录。
3. 点击编辑或修正入口。
4. 修改物流单号、UPC、IMEI 或 Serial。
5. 点击保存。
6. 系统同步更新入库记录和关联库存。
7. 如果该记录属于强制入库，编辑框会显示 `补货 / 强制入库留言`；保存后新留言会覆盖
   原 `forceReason`，刷新页面和客户库存后仍然保留。

强制入库：

1. 找到状态为异常且确实应该入库的记录。
2. 点击 `强制入库`。
3. 输入原因。
4. 系统重新检查 UPC 和 IMEI/Serial。
5. 校验通过后生成库存并解决相关异常。
6. 原因会在 `客户库存` 明细的 `补货留言` 中直接显示，例如 `chen补给JH`。

注意事项：

- 强制入库不能绕过 UPC 匹配。
- 强制入库不能绕过 IMEI/Serial 唯一性。
- 已装箱或已出库库存不能通过入库记录静默修改。
- 修正和强制入库都会写入审计日志。

## 10. 客户库存

`客户库存` 用于查看客户当前库存、SKU 汇总和单件明细。

可查看内容：

- 总库存。
- SKU 数量。
- 在库数量。
- 可出库数量。
- 已装箱数量。
- 已出库数量。
- 异常和作废数量。
- 每个 SKU 的数量汇总。
- 每台设备的 IMEI/Serial、入库批次、物流单号、最新箱号、补货留言和状态。

查看指定客户：

1. 点击左侧 `客户库存`。
2. 选择客户和仓库。
3. 查看汇总卡片。
4. 点击 `在库`、`可出库`、`已装箱`、`已出库`、`异常` 等卡片切换明细。
5. 搜索单号、入库单、箱号、IMEI、UPC 或商品名称定位货物。

查未知货物：

1. 客户选择 `全部客户`。
2. 输入单号、IMEI、Serial、UPC、SKU、商品名或客户名。
3. 查看结果中的客户列。
4. 后续需要出库装箱时，切回具体客户。

注意事项：

- `全部客户` 主要用于查找归属。
- 实际装箱必须选择具体客户和仓库。
- 客户库存页面不负责创建出库箱。

## 11. 出库装箱

`出库装箱` 用于把客户库存加入出库箱。系统要求选择客户和仓库，且只能装入同客户、同仓库、`IN_STOCK` 状态的库存。

### 11.1 基本规则

- 出库不能改变库存客户归属。
- `全部客户` 模式只用于查询，不能创建箱子或装箱。
- 只有 `IN_STOCK` 库存可以装箱。
- 加入箱子后库存变为 `PACKED`。
- 封箱必须上传至少一张照片或一段视频。

### 11.2 创建或选择箱子

1. 点击左侧 `出库装箱`。
2. 选择客户和仓库。
3. 点击新建箱子。
4. 系统生成箱子名称。
5. 如已有打开状态箱子，可从已创建箱子列表选择继续装箱。

箱子名称格式类似：

```text
客户名称YYYYMMDD箱1
```

内部箱号由系统维护，用于唯一性和报表追踪。

### 11.3 新建箱子弹窗编辑

新建或打开箱子后，弹出的明细中可以直接修改箱子信息：

- 箱子名称。
- 尺寸。
- 重量。
- 出库物流号 / 面单号。
- 备注。

保存规则：

- 箱子未封箱时可以编辑。
- 已封箱箱子需要先重开为 `OPEN`，再修改。
- 固定尺寸可选择 `12*12*12`、`16*16*12`、`18*18*12`、`18*18*16`。
- 特殊尺寸选择自定义后填写尺寸。
- 清空备注或单号后保存，会真正清空该字段。
- 全选或拖动选中箱名文字不会关闭弹窗；只有点击明确的关闭/取消控件才退出。

### 11.4 逐一扫码装箱

1. 确认当前箱子为 `OPEN`。
2. 扫描 UPC。
3. 扫描 IMEI 或 Serial。
4. 系统确认 UPC 和 IMEI/Serial 指向同一条可装箱库存。
5. 成功后自动加入当前箱。
6. 继续扫描下一件。

扫码异常时：

- UPC 和 IMEI/Serial 不属于同一条库存，系统会暂停。
- 货物不存在、客户不一致、仓库不一致、已经装箱或状态不可用，系统会阻止装箱。
- 操作员需要清空或修正扫描值后再继续。

### 11.5 批量装箱

1. 选择客户和仓库。
2. 切换到批量装箱相关区域。
3. 按成色、品类、搜索条件筛选库存。
4. 勾选要装箱的货物；如果不勾选，系统使用当前筛选结果。
5. 输入箱子数量和每箱数量。
6. 确认每箱数量合计等于当前选择或筛选总数。
7. 在预览里检查每箱明细和箱名。
8. 如需要，直接修改每个计划箱子的可见箱名。
9. 提交后系统创建箱子并把库存加入箱子。

注意事项：

- 批量装箱不会创建库存，只会把已有库存加入箱子。
- 改客户或改仓库会清空当前选择，避免混装。
- 批量筛选按当前客户和仓库的可用库存计算，不只看当前页。

### 11.6 箱子明细、打印和下载

打开箱子后可执行：

- 查看箱内明细。
- 编辑箱子基础信息。
- 打印箱内明细。
- 下载当前箱装箱明细。
- 从箱子移除单件货物。
- 清空箱内全部货物。
- 上传或删除装箱照片、视频。
- 封箱。

单件货物点击 `删除` 后直接退回客户库存，不再弹出第二次确认。整箱和批量
删除仍保留确认保护。打印预览首行显示当前完整箱子名称及箱子创建时间，时间按当前
仓库时区显示为 `MM.DD HH:mm`；不再使用客户名称，也不会另外重复打印简短箱号。

已创建箱子列表可执行：

- 勾选一个或多个箱子。
- 下载选中箱子数据。
- 下载当前筛选范围内的全部箱子数据。
- 切换 `未完成`、`已封箱`、`全部历史`。

### 11.7 封箱

1. 打开要封箱的箱子。
2. 检查箱内明细和数量。
3. 上传至少一张照片或一段视频。
4. 点击封箱。
5. 系统把箱子状态改为 `SEALED`，并写入审计日志。

封箱后当前阶段库存仍保持 `PACKED`。最终 `OUTBOUND` 状态留给后续最终出库流程。

### 11.8 返工和删除

- 已封箱箱子如需返工，可重开为 `OPEN`。
- 重开后可以继续加货、移除货物、更新证据，再重新封箱。
- 删除打开状态箱子时，箱子会作废，箱内货物退回 `IN_STOCK`。
- 已封箱箱子需要先重开，才能删除。

### 11.9 回到顶部

装箱页面向下滚动后，会出现 `回到顶部` 按钮。点击后页面回到顶部，方便继续选择客户、创建箱子或扫码装箱。

## 12. 异常池

`异常池` 集中处理扫码、入库、库存和出库过程中出现的问题。

常见异常：

- `UPC_NOT_MATCHED`：UPC 未匹配启用商品。
- `IMEI_DUPLICATED`：IMEI 或 Serial 重复。
- `UPS_DUPLICATED`：物流单号重复。
- `CUSTOMER_OWNERSHIP_MISMATCH`：客户归属不一致。
- `IMEI_NOT_INBOUNDED`：引用了未入库的 IMEI。

处理步骤：

1. 点击左侧 `异常池`。
2. 按客户、异常类型或状态筛选。
3. 点击异常行查看详情。
4. 根据复核结果选择解决、忽略或作废。
5. 填写处理说明。
6. 提交处理。

注意事项：

- 只有 `OPEN` 异常可以处理。
- 处理异常必须填写说明。
- 异常处理会写入审计日志。

## 13. 批量修改客户

`批量修改客户` 用于纠正入库时选错客户的历史错误。它会同步修改入库记录、库存归属和相关异常归属，并保存客户变更日志。

适用场景：

- 已确认入库后发现客户选错。
- 库存还没有装箱或出库。
- 需要保留完整修改原因和审计记录。

不适用场景：

- 库存已经 `PACKED`。
- 库存已经 `OUTBOUND`。
- 只是想在出库时临时换客户。

操作步骤：

1. 点击左侧 `批量修改客户`。
2. 选择当前客户。
3. 选择新客户。
4. 按仓库、时间、单号、UPC、IMEI、商品名或搜索词筛选候选记录。
5. 勾选需要修改的入库记录。
6. 点击预览。
7. 检查影响入库项、库存项、异常记录和阻断项。
8. 填写修改原因。
9. 提交修改。
10. 到 `客户库存` 确认归属已更新。

注意事项：

- 提交前必须先预览。
- 如果预览后记录发生变化，提交会被拒绝，需要重新预览。
- 新客户必须启用，且不能和当前客户相同。
- 修改原因必填。

## 14. 明细下载

`明细下载` 用于生成入库、装箱、库存、异常、客户修改日志和审计日志等报表文件。

支持报表：

- 入库明细。
- 装箱明细。
- 库存明细。
- 异常明细。
- 客户修改日志。
- 审计日志。

通用下载流程：

1. 点击左侧 `明细下载`。
2. 选择报表类型。
3. 选择客户、仓库、时间范围、状态或搜索词。
4. 点击 `预览`。
5. 查看预计行数和样例数据。
6. 点击生成导出。
7. 在导出历史里下载文件。

入库明细下载：

1. 报表类型选择 `入库明细`。
2. 可选择具体入库批次。
3. 预览后生成导出。
4. 文件名会包含批次号，便于查找。

装箱明细下载：

1. 报表类型选择 `装箱明细`。
2. 可按客户、时间、搜索词、封箱状态筛选箱子。
3. 勾选需要导出的箱子。
4. 预览并生成 Excel。
5. 在导出历史下载。

留仓库存下载：

1. 报表类型选择 `库存明细`。
2. 格式选择 Excel。
3. 布局选择留仓汇总。
4. 系统导出 `IN_STOCK` 留仓库存。

注意事项：

- 修改筛选条件后，需要重新预览。
- 结束时间不能早于开始时间。
- 导出记录只允许创建人下载。
- 报表导出会写入审计日志。

## 15. 包裹预报和包裹预警

当前生产版本默认关闭 `包裹预报` 和 `包裹预警` 菜单。

当前入库流程仍按手动选择客户并锁定客户执行，不会根据预报单号自动带出客户。

如果后续重新启用预报功能，需要管理员同时打开后端和前端功能开关，并重新部署系统。

## 16. 常见业务场景

### 新客户第一次入库

1. `客户管理` 新增客户。
2. `商品管理` 确认 UPC 已存在且启用。
3. `入库扫码` 选择客户并锁定。
4. 扫描物流单号、UPC、IMEI / Serial。
5. 复核当前入库单。
6. 点击 `确认入库`。
7. 到 `客户库存` 搜索 IMEI 或单号确认库存已生成。

### 找不到某台机器属于哪个客户

1. 到 `客户库存`。
2. 客户选择 `全部客户`。
3. 搜索 IMEI、Serial、单号、UPC 或商品名。
4. 查看客户列确认归属。
5. 如果要装箱，切回具体客户。

### 入库时 UPC 不匹配

1. 系统会在扫码阶段拒绝加入明细。
2. 到 `商品管理` 查询 UPC。
3. 如果商品资料缺失，先新增或导入正确 UPC。
4. 回到 `入库扫码` 重新扫描。

### 入库后发现客户选错

1. 确认该库存还没有装箱或出库。
2. 到 `批量修改客户`。
3. 选择原客户和新客户。
4. 筛选并勾选错误记录。
5. 预览影响。
6. 填写原因并提交。
7. 到 `客户库存` 确认归属已更新。

### 已装箱后需要返工

1. 到 `出库装箱`。
2. 选择客户和仓库。
3. 找到已封箱箱子。
4. 重开箱子。
5. 添加、移除或修正箱内货物。
6. 更新照片或视频证据。
7. 重新封箱。

### 给客服下载某几个箱子的明细

1. 到 `明细下载`。
2. 选择 `装箱明细`。
3. 用客户、日期、搜索词或封箱状态找到箱子。
4. 勾选需要导出的箱子。
5. 预览。
6. 生成 Excel 导出。
7. 在导出历史下载文件。

## 17. 新人必须记住的规则

1. 入库前先锁定客户，出库不能改客户。
2. UPC 决定商品，IMEI/Serial 决定单件追踪。
3. 随机物流单号、未维护 UPC、重复 IMEI/Serial 不能加入待入库明细。
4. 待入库明细在确认前可以编辑、保存、取消或删除。
5. 异常行必须先修正或删除，不能带着异常继续扫。
6. 确认入库必须人工点击，文件导入不会自动入库。
7. 只有 `IN_STOCK` 库存可以装箱。
8. 装箱后库存变成 `PACKED`。
9. 封箱必须上传照片或视频。
10. 入库后客户选错，用 `批量修改客户`。
11. 查未知货物，优先用 `全部客户` 搜索；实际操作必须切回具体客户。
12. 报表导出前一定先预览。

## 19. 出问题时先看哪里

| 问题             | 优先查看页面                 | 处理方向                                     |
| ---------------- | ---------------------------- | -------------------------------------------- |
| 扫 UPC 不识别    | 商品管理、入库扫码           | 检查 UPC 是否存在且启用                      |
| IMEI/Serial 重复 | 入库扫码、入库记录、客户库存 | 搜索 IMEI/Serial，确认是否已入库             |
| 物流单号不能加入 | 入库扫码、入库记录           | 检查是否符合 UPS、USPS、FedEx 或 BB0000 规则 |
| 不知道货属于谁   | 客户库存、出库装箱           | 使用全部客户搜索                             |
| 不能装箱         | 出库装箱、客户库存           | 检查客户、仓库、状态是否为 IN_STOCK          |
| 不能封箱         | 出库装箱                     | 检查箱内是否有货、是否上传照片或视频         |
| 箱子尺寸保存报错 | 出库装箱                     | 使用预设尺寸或自定义尺寸保存                 |
| 客户选错         | 批量修改客户                 | 预览后填写原因提交                           |
| 报表数据不对     | 明细下载、客户库存           | 重新检查筛选条件并预览                       |
| 操作追责         | Dashboard、审计日志报表      | 查看最近操作和导出审计日志                   |
