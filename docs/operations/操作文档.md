# WMS Scan 系统操作手册

本文面向新入职仓库操作员、客服对账人员和主管，用来快速了解 WMS Scan 的功能入口、日常操作顺序和常见处理方法。

## 1. 系统定位

WMS Scan 用于美国仓库 Apple 产品的扫码入库、客户库存管理、出库装箱、异常处理、批量客户修正、明细下载和审计追踪。

核心业务链路：

1. 先维护客户、商品 UPC 和仓库设置。
2. 入库时先选择并锁定客户，再扫描物流单号、UPC、IMEI 或 Serial。
3. 确认入库后，系统生成该客户名下的库存，状态为 `IN_STOCK`。
4. 出库时只能操作同客户、同仓库、`IN_STOCK` 的库存。
5. 装箱后库存变为 `PACKED`，封箱需要上传照片或视频证据。
6. 异常、客户变更、入库确认、封箱和报表导出都会留下审计记录。

## 2. 新人上手顺序

1. 登录系统，确认左侧菜单和当前账号。
2. 到 `客户管理` 查看客户编码和客户名称。
3. 到 `商品管理` 查看 SKU、UPC、商品名称和 IMEI 要求。
4. 到 `扫码入库` 跑一单测试入库，理解锁定客户、扫码、复核、确认入库。
5. 到 `客户库存` 查刚入库的 IMEI 或单号。
6. 到 `出库装箱` 创建箱子、扫码或批量装箱、上传证据、封箱。
7. 到 `明细下载` 按入库批次、库存或箱子导出文件。
8. 到 `异常池` 和 `入库记录` 学习错误数据的修正路径。

## 3. 登录和账号

### 功能介绍

登录用于识别当前操作员。所有关键操作都会记录操作员、时间、动作和修改前后内容。

### 使用方法

1. 打开系统地址。
2. 输入邮箱和密码。
3. 点击 `登录`。
4. 登录后左侧显示功能菜单，顶部显示当前用户。
5. 页面数据不是最新时，点击顶部 `刷新数据`。
6. 操作结束后点击 `退出`。

### 注意事项

- 不要共用账号，否则审计日志无法区分是谁操作。
- 账号被停用后不能继续登录。
- 登录过期时，系统会提示重新登录。

## 4. Dashboard

### 功能介绍

`Dashboard` 是日常运营看板，用来快速查看仓库当天情况和最近操作。

主要展示：

- 今日入库数量。
- 今日封箱数量。
- 当前在库数量。
- 待处理异常数量。
- 最近 7 天入库、出库、异常趋势。
- 最近审计操作记录。

### 使用方法

1. 点击左侧 `Dashboard`。
2. 查看今日入库、待处理异常和库存概览。
3. 刚完成入库、封箱或异常处理后，可点击顶部 `刷新数据`。
4. 待处理异常数量不为 0 时，进入 `异常池` 处理。

## 5. 客户管理

### 功能介绍

客户是库存归属的基础。入库、库存、出库、报表和批量客户变更都围绕客户展开。

客户管理用于：

- 新增客户。
- 查看客户列表。
- 编辑客户编码和客户名称。
- 查看客户相关库存和业务汇总。

### 使用方法

1. 点击左侧 `客户管理`。
2. 新增客户时填写 `客户编码` 和 `客户名称`。
3. 点击 `新增客户`。
4. 修改客户时，在客户列表里点击 `编辑`。
5. 修改编码或名称后点击 `保存`。

### 注意事项

- 客户编码是稳定业务标识，系统会转成大写。
- 客户不要随意删除；不再使用时应停用，历史记录仍需要保留客户归属。
- 入库时选错客户，不能在出库时改客户，应走 `批量修改客户`。

## 6. 商品管理

### 功能介绍

`商品管理` 维护 UPC 商品库。入库扫描 UPC 时，系统通过 UPC 匹配商品信息，包括 SKU、型号、颜色、容量和是否要求 IMEI。

商品管理用于：

- 新增单个商品和 UPC。
- 查询 UPC 对应商品。
- 下载商品导入模板。
- 批量导入商品。

### 使用方法

单个新增：

1. 点击左侧 `商品管理`。
2. 填写 `SKU`、商品名称、型号编码和 UPC。
3. 点击新增。

批量导入：

1. 点击 `下载导入模板`。
2. 按模板填写 `sku`、`name`、`brand`、`model`、`category`、`color`、`capacity`、`requiresImei`、`upcs`。
3. 一个商品多个 UPC 时，用英文分号 `;` 分隔。
4. 上传文件并确认导入。

UPC 查询：

1. 在 UPC 查询框输入 UPC。
2. 查询后确认是否匹配到正确商品。

### 注意事项

- UPC 必须是数字，并且全系统唯一。
- SKU 必须唯一，系统会转成大写。
- 入库扫描时只能匹配启用状态的商品和 UPC。
- 商品需要 IMEI 时，入库必须提供合法 IMEI 或系统认可的 Apple 设备标识。

## 7. 系统设置

### 功能介绍

`系统设置` 用于维护默认仓库、扫码规则、异常生成规则、通知开关和数据保留天数。

常见设置包括：

- 默认仓库。
- 入库是否必须锁定客户。
- 出库是否强制客户归属一致。
- 是否检查重复 IMEI。
- 是否检查重复物流单号。
- 是否自动生成未匹配 UPC、重复 IMEI、重复物流单号异常。
- 审计、报表、异常数据保留天数。

### 使用方法

1. 点击左侧 `系统设置`。
2. 选择默认仓库。
3. 按业务要求开关扫码规则和异常规则。
4. 设置数据保留天数。
5. 点击保存。

### 注意事项

- 系统设置会影响后续入库、出库和异常处理，普通操作员不要随意修改。
- 修改设置会写入审计日志。

## 8. 扫码入库

### 功能介绍

`扫码入库` 是收货的核心页面。操作员先锁定客户，再扫描物流单号、UPC、IMEI 或 Serial，最后人工复核并点击确认入库。

入库支持两种模式：

- `一版模式`：物流单号、UPC、IMEI 都扫入后自动加入当前入库单。
- `物流+UPC 模式`：物流单号和 UPC 扫入后自动加入当前入库单，适合暂不录 IMEI 的场景，但确认时仍受商品规则限制。

支持的物流单号规则：

- UPS 常规单号。
- 以 `9622` 开头、总长度 22 到 34 位的 FedEx 单号。
- 以 `BB0000` 开头的仓库赔偿包裹号。
- USPS、非 9622 FedEx 或其他格式会提示异常，需要人工确认后才能继续。

### 标准操作流程

1. 点击左侧 `入库扫码`。
2. 选择客户。
3. 点击 `锁定客户`，创建当前入库草稿。
4. 选择入库模式。
5. 扫描或输入物流单号。
6. 扫描或输入 UPC。
7. 按商品规则扫描或输入 IMEI。
8. 系统自动加入明细，或点击 `加入明细`。
9. 在 `当前入库单` 里检查件数、UPC、商品、物流单号和异常。
10. 确认无误后点击 `确认入库`。
11. 系统生成库存，库存状态为 `IN_STOCK`。

### 批量文件导入

1. 在 `扫码入库` 下载入库模板。
2. 按模板填写 `单号`、`upc`、`imei`。
3. 上传 CSV 文件。
4. 系统把文件内容加入当前草稿，不会自动确认入库。
5. 复核当前入库单。
6. 点击 `确认入库`。

### 异常和修正

入库草稿中如果出现异常行，系统会暂停继续入库。常见原因：

- UPC 未匹配商品。
- IMEI 或 Serial 重复。
- 物流单号格式异常或历史重复。

处理方法：

1. 在当前入库单找到异常行。
2. 点击 `修正`。
3. 修改物流单号、UPC、IMEI 或 Serial。
4. 保存后系统重新校验。
5. 如果该行确实不要入库，可删除该草稿行。
6. 异常行修正或删除后，才能继续扫描或确认入库。

### 注意事项

- 入库客户一旦锁定，草稿内不能更换客户。
- 最终入库必须人工点击 `确认入库`，系统不会因为扫码或导入自动入库。
- 同一个草稿内重复 IMEI 或 Serial 会阻止确认。
- 已确认入库后发现客户选错，应使用 `批量修改客户`，不要在出库环节改客户。

## 9. 入库记录

### 功能介绍

`入库记录` 用于查询已确认入库或异常入库明细，并处理确认后的错误。

可以处理：

- 查询入库明细。
- 按客户、状态、单号、UPC、IMEI 等筛选。
- 修正已确认入库的物流单号、UPC、IMEI 或 Serial。
- 对异常但经人工复核可入库的记录执行强制入库。

### 使用方法

查询记录：

1. 点击左侧 `入库记录`。
2. 选择客户、状态或输入搜索条件。
3. 查看入库明细。

修正入库货物：

1. 找到需要修正的记录。
2. 点击编辑或修正入口。
3. 修改物流单号、UPC、IMEI 或 Serial。
4. 点击 `保存入库`。
5. 系统会同步更新入库明细和关联库存。

强制入库：

1. 找到状态为异常且确实应该入库的记录。
2. 点击 `强制入库`。
3. 输入强制入库原因。
4. 系统重新检查 UPC 和 IMEI/Serial 重复。
5. 校验通过后生成库存并解决相关异常。

### 注意事项

- 强制入库不能绕过 UPC 匹配和 IMEI/Serial 唯一性。
- 已经装箱或出库的库存不能通过入库记录静默修改。
- 修正和强制入库都会写入审计日志。

## 10. 客户库存

### 功能介绍

`客户库存` 用于查看客户当前库存、SKU 汇总和 IMEI/Serial 明细。

可以查看：

- 总库存。
- SKU 数量。
- 在库数量。
- 可出库数量。
- 已装箱数量。
- 已出库数量。
- 异常和作废数量。
- 每个 SKU 的数量汇总。
- 每台设备的 IMEI/Serial、入库批次、物流单号、最新箱号和状态。

### 使用方法

查看指定客户库存：

1. 点击左侧 `客户库存`。
2. 选择客户和仓库。
3. 查看上方汇总卡片。
4. 点击 `在库`、`可出库`、`已装箱`、`已出库`、`异常` 等卡片，可切换下方明细。
5. 在搜索框输入单号、入库单、箱号、IMEI、UPC 或商品名称定位具体货物。

跨客户查找未知货物：

1. 客户选择 `全部客户`。
2. 输入单号、IMEI、Serial、UPC、SKU、商品名或客户名。
3. 查看结果中的客户列，确定货物归属。
4. 需要后续操作时，切回具体客户。

按日期查看：

1. 选择业务日期。
2. 系统会按状态使用对应日期过滤。
3. `PACKED` 看装箱时间，`OUTBOUND` 看出库时间，其他库存看入库时间。

### 注意事项

- 客户库存页面主要用于查看和清理库存，不负责创建出库箱。
- 出库装箱必须到 `出库装箱` 页面操作。
- `全部客户` 视图用于查找归属，不用于混合客户装箱或删除。

## 11. 出库装箱

### 功能介绍

`出库装箱` 用于把客户库存加入出库箱。系统要求选择客户和仓库，且只能装入同客户、同仓库、`IN_STOCK` 状态的库存。

出库支持两种方式：

- 逐一扫码装箱：扫描 UPC 和 IMEI/Serial，系统匹配同一条库存后加入当前箱。
- 批量装箱：筛选库存、勾选货物、按每箱数量生成多个箱子。

箱子名称由系统生成，格式类似：

```text
客户名称YYYYMMDD箱1
```

### 创建或选择箱子

1. 点击左侧 `出库装箱`。
2. 选择具体客户和仓库。
3. 点击新建箱子，系统生成箱名。
4. 如已有打开状态箱子，可从已创建箱子列表选择继续装箱。

### 逐一扫码装箱

1. 确认当前箱子为 `OPEN`。
2. 在扫码输入框扫描 UPC。
3. 再扫描 IMEI 或 Serial。
4. 系统确认 UPC 和 IMEI/Serial 指向同一条可装箱库存。
5. 成功后自动加入当前箱。
6. 继续扫描下一件。

扫码异常时：

- UPC 和 IMEI/Serial 不属于同一条库存，系统会暂停。
- 货物不存在、客户不一致、仓库不一致、已经装箱或状态不可用，系统会阻止装箱。
- 操作员需要清空或修正扫描值后再继续。

### 批量装箱

1. 在出库装箱页选择客户和仓库。
2. 可按成色、品类、搜索条件筛选库存。
3. 勾选要装箱的货物；如果不勾选，系统使用当前筛选结果。
4. 输入箱子数量和每箱数量。
5. 确认每箱数量合计等于当前选择或筛选总数。
6. 在预览里检查每箱明细和箱名。
7. 提交后系统创建箱子并把库存加入箱子。

### 箱子编辑、打印和下载

打开箱子时可执行：

- 编辑箱名、尺寸、重量、备注。
- 添加或修改出库物流号、标签号。
- 查看箱内明细。
- 打印箱内明细。
- 下载当前箱装箱明细。
- 从箱子移除单件或清空全部货物。
- 上传或删除装箱照片、视频。

已创建箱子列表可：

- 勾选一个或多个箱子。
- 点击 `下载选中数据`，导出只包含这些箱子的装箱明细。
- 点击 `下载全部数据`，导出当前客户和仓库范围内的装箱明细。

### 封箱

1. 打开要封箱的箱子。
2. 检查箱内明细和数量。
3. 上传至少一张照片或一段视频作为装箱证据。
4. 点击封箱。
5. 系统把箱子状态改为 `SEALED`，并写入审计日志。

### 返工和删除

- 已封箱箱子如需返工，可重开为 `OPEN`。
- 重开后可以继续加货、移除货物、更新证据，再重新封箱。
- 删除打开状态箱子时，箱子会作废，箱内货物退回 `IN_STOCK`。
- 已封箱箱子需要先重开，才能删除。

### 全部客户查找

1. 客户选择 `全部客户`。
2. 输入单号、IMEI/Serial、客户、UPC、SKU 或商品关键词。
3. 查看客户、状态和所在箱。
4. 如果要实际装箱，必须切换到该货物所属的具体客户。

### 注意事项

- `全部客户` 模式只读，不能装箱、建箱、批量装箱或扫码装箱。
- 出库不能改变库存客户归属。
- 只有 `IN_STOCK` 库存可装箱。
- 封箱必须有照片或视频证据。

## 12. 异常池

### 功能介绍

`异常池` 集中处理扫码、入库、库存和出库过程中出现的问题。

常见异常：

- `UPC_NOT_MATCHED`：UPC 未匹配启用商品。
- `IMEI_DUPLICATED`：IMEI 或 Serial 重复。
- `UPS_DUPLICATED`：物流单号已出现在确认入库记录中。
- `CUSTOMER_OWNERSHIP_MISMATCH`：客户归属不一致。
- `IMEI_NOT_INBOUNDED`：引用了未入库的 IMEI。

异常状态：

- `OPEN`：待处理。
- `RESOLVED`：已解决。
- `IGNORED`：已忽略但保留记录。
- `INVALID`：异常无效或不适用。

### 使用方法

1. 点击左侧 `异常池`。
2. 按客户、异常类型或状态筛选。
3. 点击异常行查看详情。
4. 根据复核结果选择处理动作。
5. 填写处理说明。
6. 提交处理。

### 注意事项

- 只有 `OPEN` 异常可以处理。
- 处理异常必须填写说明。
- 异常处理会写入审计日志。

## 13. 批量修改客户

### 功能介绍

`批量修改客户` 用于纠正入库时选错客户的历史错误。它会同步修改入库记录、库存归属和相关异常归属，并保存客户变更日志。

适用场景：

- 已确认入库后发现客户选错。
- 库存还没有装箱或出库。
- 需要保留完整修改原因和审计记录。

不适用场景：

- 库存已经 `PACKED`。
- 库存已经 `OUTBOUND`。
- 只是想在出库时临时换客户。

### 使用方法

1. 点击左侧 `批量修改客户`。
2. 选择当前客户。
3. 选择新客户。
4. 按仓库、时间、单号、UPC、IMEI、商品名或搜索词筛选候选记录。
5. 勾选需要修改的入库记录。
6. 点击预览。
7. 检查影响入库项、库存项、异常记录和阻断项。
8. 填写修改原因。
9. 提交修改。
10. 在页面下方查看客户修改日志。

### 注意事项

- 提交前必须先预览。
- 如果预览后记录发生变化，提交会被拒绝，需要重新预览。
- 新客户必须是启用状态，且不能和当前客户相同。
- 修改原因必填。

## 14. 明细下载

### 功能介绍

`明细下载` 用于生成入库、装箱、库存、异常、客户修改日志和审计日志等报表文件，便于客服对账、客户核对和内部追踪。

支持的报表类型：

- `入库明细`：每件入库货物的客户、仓库、商品、扫码和库存状态。
- `装箱明细`：每件已装箱货物的箱子、客户、商品、单号、装箱和封箱时间。
- `库存明细`：当前库存明细。
- `异常明细`：异常类型、状态、原始值、客户和处理信息。
- `客户修改日志`：批量客户变更记录。
- `审计日志`：关键操作日志。

### 通用下载流程

1. 点击左侧 `明细下载`。
2. 选择报表类型。
3. 选择客户、仓库、时间范围、状态或搜索词。
4. 按页面提示选择字段或格式。
5. 点击 `预览`。
6. 查看预计行数和样例数据。
7. 点击生成导出。
8. 在导出历史里下载已完成文件。

### 入库明细下载

1. 报表类型选择 `入库明细`。
2. 可选择具体入库批次。
3. 点击预览。
4. 生成导出。
5. 按批次导出时，文件名会包含批次号，便于查找。

### 装箱明细下载

1. 报表类型选择 `装箱明细`。
2. 默认建议选择 `仅已封箱`，避免导出还在编辑的箱子。
3. 可通过客户、搜索、开始时间、结束时间、封箱状态筛选候选箱子。
4. 勾选一个或多个箱子。
5. 点击预览。
6. 生成选中箱子导出。
7. 如果要导出选中箱子的全部装箱明细，清空时间范围后再预览导出。

Excel 装箱明细会使用客户核对表排版，适合发给客户或客服核对。CSV 则按选择字段导出普通明细。

### 留仓库存下载

1. 报表类型选择 `库存明细`。
2. 格式选择 Excel。
3. 布局选择 `留仓汇总表格`。
4. 系统只导出 `IN_STOCK` 留仓库存。
5. 导出文件会按每 24 台生成 `留仓箱1`、`留仓箱2` 等虚拟箱号。

### 注意事项

- 修改筛选条件后，需要重新预览再导出。
- 结束时间不能早于开始时间。
- 导出记录只允许创建人下载。
- 报表导出会写入审计日志。

## 15. 常见业务场景

### 新客户第一次入库

1. `客户管理` 新增客户。
2. `商品管理` 确认 UPC 已存在且启用。
3. `扫码入库` 选择客户并锁定。
4. 扫描物流单号、UPC、IMEI。
5. 复核当前入库单。
6. 点击 `确认入库`。
7. `客户库存` 搜索 IMEI 或单号确认库存已生成。

### 找不到某台机器属于哪个客户

1. 到 `客户库存`。
2. 客户选择 `全部客户`。
3. 搜索 IMEI、Serial、单号、UPC 或商品名。
4. 查看客户列确认归属。
5. 如果是出库场景，也可在 `出库装箱` 的 `全部客户` 模式查询所在箱和状态。

### 入库时 UPC 不匹配

1. 当前入库单会出现异常行并暂停。
2. 到 `商品管理` 查询 UPC 是否存在。
3. 如果商品资料缺失，先新增或导入正确 UPC。
4. 回到 `扫码入库` 修正异常行。
5. 保存后重新校验。
6. 异常解除后继续入库。

### 入库后发现客户选错

1. 确认该库存还没有装箱或出库。
2. 到 `批量修改客户`。
3. 选择原客户和新客户。
4. 筛选并勾选错误记录。
5. 预览影响。
6. 填写原因并提交。
7. 到 `客户库存` 确认归属已更新。

### 已装箱后需要返工

1. 到 `出库装箱`。
2. 选择客户和仓库。
3. 找到已封箱箱子。
4. 重开箱子。
5. 添加、移除或修正箱内货物。
6. 更新照片或视频证据。
7. 重新封箱。

### 给客服下载某几个箱子的明细

1. 到 `明细下载`。
2. 选择 `装箱明细`。
3. 用客户、日期、搜索词或封箱状态找到箱子。
4. 勾选需要导出的箱子。
5. 预览。
6. 生成 Excel 导出。
7. 在导出历史下载文件。

## 16. 新人必须记住的规则

1. 入库前先锁定客户，出库不能改客户。
2. UPC 决定商品，IMEI/Serial 决定单件追踪。
3. 异常行必须先修正或删除，不能带着异常继续扫。
4. 确认入库必须人工点击，文件导入不会自动入库。
5. 只有 `IN_STOCK` 库存可以装箱。
6. 装箱后库存变成 `PACKED`。
7. 封箱必须上传照片或视频。
8. 入库后客户选错，用 `批量修改客户`。
9. 查未知货物，优先用 `全部客户` 搜索，但实际操作必须切回具体客户。
10. 报表导出前一定先预览。

## 17. 角色分工建议

### 仓库入库员

主要使用：

- `扫码入库`
- `入库记录`
- `客户库存`
- `异常池`

重点掌握：锁定客户、扫描顺序、草稿异常修正、确认入库前复核。

### 仓库出库员

主要使用：

- `客户库存`
- `出库装箱`
- `明细下载`

重点掌握：同客户同仓库装箱、逐一扫码和批量装箱、上传证据、封箱和返工。

### 客服或对账人员

主要使用：

- `客户库存`
- `明细下载`
- `入库记录`
- `出库装箱` 的箱子明细和下载。

重点掌握：按客户、单号、IMEI、箱号查货，按入库批次或选中箱子下载明细，导出留仓库存。

### 主管或管理员

主要使用：

- `客户管理`
- `商品管理`
- `系统设置`
- `批量修改客户`
- `异常池`
- `Dashboard`

重点掌握：基础资料维护、错误客户归属修正、异常处理审核、审计日志追踪。

## 18. 出问题时先看哪里

| 问题           | 优先查看页面                 | 处理方向                             |
| -------------- | ---------------------------- | ------------------------------------ |
| 扫 UPC 不识别  | 商品管理、扫码入库           | 检查 UPC 是否存在且启用              |
| IMEI 重复      | 扫码入库、入库记录、客户库存 | 搜索 IMEI，确认是否已入库            |
| 不知道货属于谁 | 客户库存、出库装箱           | 使用全部客户搜索                     |
| 不能装箱       | 出库装箱、客户库存           | 检查客户、仓库、状态是否为 IN_STOCK  |
| 不能封箱       | 出库装箱                     | 检查箱内是否有货、是否上传照片或视频 |
| 客户选错       | 批量修改客户                 | 预览后填写原因提交                   |
| 报表数据不对   | 明细下载、客户库存           | 重新检查筛选条件并预览               |
| 操作追责       | Dashboard、审计日志报表      | 查看最近操作和导出审计日志           |
